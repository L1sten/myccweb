# 传播投手 - 需求管理系统 PRD

## 1. 产品概述

传播投手是一款面向会展中心及活动主办方的传播需求管理工具。运营管理员通过后台管理需求、匹配方案并协调执行。

- **运营管理员**：通过管理后台处理和管理传播需求（微博热搜、抖音热推、稿件推送、央媒传播、定制传播）

---

## 2. 管理后台 - 需求列表

### 2.1 页面结构

- 顶部导航栏：Logo、产品名称、页面标题、通知铃铛、用户信息
- 页面标题区：标题、需求总数、刷新和导出按钮
- 筛选栏：搜索框 + 服务类型标签筛选
- 数据表格：编号、需求名称、服务类型、提交公司、提交时间、操作
- 分页底栏

### 2.2 搜索与筛选

- **展示规则**：搜索框支持按需求名称、公司、联系人、编号模糊匹配；服务类型标签支持单选切换
- **交互规则**：搜索和筛选条件取交集，实时更新表格
- **取数规则**：从需求管理数据库中读取全部需求记录，包含编号、需求名称、联系人、提交公司、服务类型、提交时间等字段
- **数据筛选规则**：搜索框支持按需求名称、公司、联系人、编号进行模糊匹配（LIKE）；服务类型标签支持单选切换，6个筛选选项：全部类型、微博热搜、抖音热推、稿件热推、央媒传播、定制传播；搜索条件与服务类型筛选取交集
- **数据排序规则**：默认按提交时间倒序排列（最新提交排在最前）

### 2.3 数据表格

- **展示规则**：白色圆角表格，每行显示编号、需求名称（含联系人和时间副标题）、服务类型标签、公司、提交时间、详情链接
- **交互规则**：点击行或详情链接跳转到需求详情页
- **取数规则**：读取经过搜索与筛选后的需求列表数据，每条记录包含编号、需求名称、联系人、提交公司、服务类型、提交时间
- **数据排序规则**：按提交时间倒序排列（最新提交排在最前）

---

## 3. 管理后台 - 需求详情

### 3.1 页面结构

- 返回按钮 + 需求标题 + 编号和提交时间
- 两栏布局：左侧（基本信息、需求描述、附件材料）+ 右侧（对接人信息、需求概要）

### 3.2 基本信息

- **展示规则**：2x2网格，展示展会名称、所属公司、服务类型（蓝色高亮）、传播周期
- **交互规则**：纯展示
- **取数规则**：根据需求编号从需求管理数据库中读取该条需求的展会名称、所属公司、服务类型、传播周期

### 3.3 需求描述

- **展示规则**：纯文本展示，保留换行格式
- **交互规则**：纯展示

### 3.4 附件材料

- **展示规则**：文件列表，每行显示文件图标、文件名和下载按钮
- **交互规则**：点击下载按钮触发下载

### 3.5 对接人信息

- **展示规则**：头像（姓名首字）+ 姓名 + 公司；下方显示联系方式、职务、展会名称
- **交互规则**：纯展示
- **取数规则**：根据需求编号读取该条需求的对接人姓名、公司、联系方式、职务、展会名称

### 3.6 需求概要

- **展示规则**：蓝色渐变背景卡片，展示服务类型、明细、周期、附件数量
- **交互规则**：纯展示
- **取数规则**：根据需求编号读取该条需求的服务类型、传播明细、传播周期、附件数量

---

## 4. 全局规则

- 管理后台支持关键词搜索和服务类型筛选，条件取交集
- 表格行点击跳转详情页，详情页支持返回列表
- **取数规则**：所有页面数据均从需求管理数据库中读取，详情页根据需求编号读取单条完整记录
- **数据排序规则**：需求列表默认按提交时间倒序排列，详情页内无排序需求
