# 正能量态势感知平台优化项目 PRD

> 版本：v1.3
> 日期：2026-06-26
> 需求范围：在现有正能量态势感知平台基础上，新增「正能量矩阵」「矩阵传播分析」「正能量周报」三个模块，形成账号监测管理、传播分析、周报配置与预览闭环。

---

## 1. 产品概述

### 1.1 背景

现有正能量态势感知平台已具备正能量看板、稿件、热榜、热点日报等能力，但客户当前仍依赖运营人员手动整理《涉舟网络正能量数据简报》。人工报告存在三个核心问题：

- **统计链路分散**：稿池投稿、热点事件、账号传播表现分别依赖人工汇总，周期复用性弱。
- **账号名单口径不可落地**：原报告中的「正能量账号名单」来自话题热门参与情况，但平台无法稳定识别所有话题下的热门参与账号，且不同平台开放字段差异较大。
- **周报缺少产品化入口**：客户周报周期通常为「上周三到本周三」，样例中还存在精确到小时的统计口径，如 `6月3日20:00 至 6月10日16:00`，需要支持可配置时间范围、候选数据挑选和报告预览。

本次优化通过定向采集和管理正能量矩阵账号，分析矩阵账号在统计周期内发布的内容和传播效果，并将结果体现在周报预览中，用可采集、可核验、可复用的口径替代不可实现的「话题热门参与账号名单」。

### 1.2 目标

- 建立可维护、可采集、可核验的正能量矩阵账号管理能力。
- 支持按统计周期分析矩阵账号内容发布、传播量、互动量和重点内容表现。
- 在正能量热点报告中补充周报配置、候选数据挑选和预览能力。
- 使用「矩阵账号本周内容及传播效果」替代不稳定的「按话题自动识别全部热门参与账号」口径。

### 1.3 用户角色

- 运营人员：维护矩阵账号、管理采集状态、筛选候选数据、预览周报内容。
- 管理人员：查看矩阵运行情况、矩阵传播分析结果和周报产出情况。
- 审核 / 协同人员：在具备权限时查看账号状态变更、处理通知或审核相关操作。

### 1.4 使用场景

- 运营人员定期维护正能量矩阵账号及采集状态，确认账号是否纳入周报。
- 运营人员按周查看矩阵账号传播表现，筛选可进入周报的数据。
- 运营人员在周报设置页调整统计周期、移除或恢复候选数据，并预览当前周报内容。
- 管理人员查看周报数据口径和候选数据保留情况，确认报告数据可采集、可核验、可复用。

### 1.5 本期范围

- 新增「正能量矩阵」入口，用于定向账号管理、采集状态管理和周报纳入控制。
- 新增「矩阵传播分析」入口，用于查看矩阵账号在周期内的发文、互动、传播趋势和稿件明细。
- 在现有「正能量热点报告」模块内新增「周报」tab，用于周报候选数据挑选和预览。

### 1.6 非本期范围

- 不承诺按话题自动识别全部热门参与账号；周报表 3 改为「矩阵账号本周内容及传播效果」。
- 平台无法提供传播量字段时，统一展示「数据缺失」，不得用互动量或估算值替代。
- 不新增独立的周报一级菜单。
- 不提供周报正文独立编辑流程。
- 不对已确认的既有报告引用进行回溯改写。

### 1.7 关键约束 / 待确认问题

- 平台无法提供传播量字段时，统一展示「数据缺失」，不得用互动量或估算值替代。
- 周报默认周期按客户周报习惯配置，默认起止小时待客户确认；原型以 `16:00` 作为示例。
- 粉丝数、阅读数、传播量、互动量等关键指标缺失时统一展示「数据缺失」，不展示为 `0`，除非业务口径明确该指标真实值为 0。
- 周报候选数据存在关键字段缺失时允许预览，但预览中必须展示数据缺失说明；候选数据全部移除或统计周期无效时禁止预览。
- 正能量矩阵账号列表默认按最新操作时间倒序展示；无最新操作时间时按账号创建时间倒序展示。

## 2. 全局交互规则

### 2.1 页面入口与导航

- 正能量矩阵、矩阵传播分析、正能量热点报告入口按用户权限展示。
- 无权限用户不可见对应入口；仅具备查看权限时，仅可查看页面内容，不可执行新增、修改、删除、导出、下载等操作。
- 正能量周报作为「正能量热点报告」模块下的报告类型展示，不新增顶部一级菜单。

### 2.2 筛选与查询

- 多个筛选条件同时存在时默认取交集。
- 关键词搜索采用模糊匹配；下拉、状态、平台等枚举条件采用精确匹配。
- 点击重置后清空筛选条件、关键词和排序状态，并回到第一页。
- 筛选无结果时展示空状态，不保留上一次查询结果冒充当前结果。

### 2.3 分页与排序

- 分页、每页条数和排序切换不清空当前筛选条件。
- 用户手动排序后，分页、导出和下载使用当前排序结果。
- 各模块需明确默认排序规则；存在多级排序时需说明主排序和次级排序。

### 2.4 按钮与操作反馈

- 新增、编辑、删除、移出、导出、下载等操作需定义默认状态、可点击条件、不可点击原因、点击反馈和操作结果。
- 危险操作需二次确认。
- 操作成功后刷新受影响的数据范围；操作失败时展示失败原因，并允许用户重新操作或调整条件。
- 下载、导出类操作需使用当前筛选条件、当前统计周期和当前保留状态，不使用历史缓存条件。

### 2.5 空状态与异常状态

- 无数据时展示空状态说明，不隐藏关键区域。
- 数据缺失、采集异常、权限不足、时间范围错误、候选数据全部移除等情况需给出明确提示。
- 缺失数据不得使用估算值替代；涉及统计时需说明是否纳入汇总。
- 周报预览失败时保留当前配置状态，不清空已选择的候选数据。

### 2.6 权限与可见性

权限配置参考现有后台的树状勾选方式，以「权限点 - 操作点」控制页面入口、按钮和数据操作能力。

| 权限点 | 操作点 | 控制范围 |
|---|---|---|
| 矩阵管理 | 列表查询 | 进入正能量矩阵页面，查看账号统计、筛选区和账号列表 |
| 矩阵管理 | 矩阵管理导出 | 导出当前筛选条件下的账号列表 |
| 矩阵管理 | 通知 | 对矩阵账号相关责任人发起通知 |
| 矩阵管理 | 新增 | 使用「新建」入口新增矩阵账号 |
| 矩阵管理 | 详情 | 查看矩阵账号基础信息、采集情况和运行状态 |
| 矩阵管理 | 修改 | 编辑账号基础信息、内容定位、运行情况、是否纳入周报 |
| 矩阵管理 | 删除 | 将账号移出矩阵，不删除既有内容和既有报告引用 |
| 矩阵管理 | 审核撤回 | 撤回已提交但尚未完成处理的矩阵账号状态变更审核 |
| 矩阵传播分析 | 查看 | 进入矩阵传播分析页面，查看指标卡、趋势图、账号传播详情和稿件明细 |
| 矩阵传播分析 | 查询筛选 | 使用时间范围、平台、账号、稿件标题等筛选条件刷新分析结果 |
| 矩阵传播分析 | 导出账号明细 | 导出当前筛选条件下的账号传播详情 |
| 矩阵传播分析 | 导出稿件明细 | 导出当前筛选条件下的稿件传播明细 |
| 正能量热点报告 | 查看 | 进入正能量热点报告模块，查看自动报告、人工报告、日报和周报列表 |
| 正能量热点报告 | 日报预览 | 预览正能量热点报告日报内容 |
| 正能量热点报告 | 周报设置 | 进入周报设置页，查看统计周期和候选数据 |
| 正能量热点报告 | 周报预览 | 打开周报预览弹窗，查看总体情况、工作情况和三张表 |

权限规则：

- 权限采用树状继承逻辑：未勾选上级权限点时，下级操作点不可用。
- 仅具备查看权限时，页面可见但新增、修改、删除、导出、下载等按钮隐藏或置灰。
- 权限变更只影响后续访问控制，不改写既有报告引用和既有操作记录。

## 3. 模块一：正能量矩阵

### 3.1 模块目标与边界

正能量矩阵用于定向管理可采集、可分析、可进入周报候选池的账号资产。该模块解决「哪些账号纳入正能量矩阵」「账号是否可采集」「账号是否进入周报」三个问题。

模块边界：

- 本模块负责账号台账、运行情况、采集情况、周报纳入状态和账号操作。
- 本模块不做周期传播趋势分析，传播趋势由「矩阵传播分析」模块承载。
- 本模块不生成报告，报告列表和预览由「正能量热点报告 / 周报」模块承载。

### 3.2 页面结构

展示规则：

- 页面顶部按现有矩阵管理统计维度展示账号运行情况统计。
- 顶部导航点击「正能量矩阵」进入本模块，并展示当前导航激活态。
- 账号列表区域展示「新媒体矩阵 / 状态变更审核」tab，默认进入「新媒体矩阵」。
- 列表上方展示筛选区、关键词搜索、新建、删除、导出等操作入口。
- 列表底部展示总数、每页条数、页码、上一页、下一页和跳页控件。

交互规则：

- 点击「全部 / 自主运营 / 代运营」后，统计卡和账号列表同步刷新。
- 点击「新媒体矩阵 / 状态变更审核」tab 后，保留已选筛选条件并刷新列表。
- 点击分页、每页条数或可排序字段时，不清空当前筛选条件。
- 仅具备对应权限的用户可新增、编辑、移出矩阵或调整采集状态。

数据规则：

- 取数规则：取客户确认纳入正能量矩阵且未被移出矩阵的账号台账及采集状态。
- 数据筛选规则：运营范围、tab、下拉筛选、关键词搜索多条件取交集。
- 数据排序规则：默认按最新操作时间倒序展示；无最新操作时间时按账号创建时间倒序展示；用户手动切换排序后以用户选择为准。

### 3.3 账号运行情况统计

展示规则：

- 状态卡展示正常运行账号、停更账号、待注销账号、已注销/关停账号。
- 状态卡展示总数，并在有变化时展示较上月变化值。
- 内容定位卡展示采编账号、经营账号、内部信息发布账号。
- 无数据时展示 `0`，不隐藏卡片。

交互规则：

- 点击运营范围后，状态统计和内容定位统计同步刷新。
- 点击状态卡可作为快捷筛选条件，下方账号列表按对应运行状态过滤。
- 状态变化值只作为趋势提示，不作为跳转入口。

数据规则：

- 取数规则：统计当前筛选范围内的矩阵账号。
- 数据筛选规则：按运营范围、运行情况、内容定位取交集；「全部」不附加运营范围条件。
- 数据排序规则：状态卡按正常运行、停更、待注销、已注销/关停固定顺序展示；内容定位按采编、经营、内部信息发布固定顺序展示。
- 统计口径：较上月变化值以本月账号状态快照与上月账号状态快照对比，不回溯改写既有状态快照。

### 3.4 筛选条件与列表字段

筛选条件：

- 展示规则：筛选区展示所属部门、媒体类型、运行情况、所属圈层、采集情况、属性、是否认证、内容定位、自主情况、变更记录、是否已读和关键词搜索。
- 交互规则：下拉筛选项选择后即时生效；关键词搜索支持输入后回车或点击搜索触发；点击重置后清空筛选、关键词和排序状态，并回到第一页。
- 数据规则：下拉项采用精确匹配，关键词对账号名称、运维负责人、联系方式做模糊匹配，多条件取交集。

账号名称：

- 展示规则：列表中展示平台账号名称，超长时单行省略；名称使用可点击样式。
- 交互规则：点击账号名称查看账号基础信息、采集情况和运行状态；无查看权限时账号名称不可点击。
- 数据规则：取客户确认的矩阵账号名称；支持模糊搜索；默认不单独按名称排序。

媒体类型 / 子类型：

- 展示规则：媒体类型筛选项以现有矩阵管理系统字典为准，一级选项为微信、微博、抖音、视频号、其他；选择其他时展示头条号、百家号、人民号、知乎号、潮鸣号、客户端。
- 交互规则：选择微信、微博、抖音、视频号时直接按对应媒体类型筛选；选择其他下的子类型时，按具体子类型筛选。
- 数据规则：取平台字典和账号台账；支持精确筛选；默认不作为首要排序字段。

所属部门 / 所属圈层：

- 展示规则：展示账号归属部门和管理圈层，缺失时展示 `--`。
- 交互规则：筛选项选择后刷新列表，多条件筛选取交集。
- 数据规则：取账号台账中的组织归属；支持精确筛选；默认不参与排序。

属性 / 是否认证：

- 展示规则：属性展示单位、个人或其他；是否认证展示是/否。
- 交互规则：选择筛选项后列表实时刷新；清空筛选后恢复全部。
- 数据规则：取账号台账和平台认证信息；缺失认证信息时展示 `--`。

内容定位：

- 展示规则：展示采编、经营、内部信息发布等定位；多个定位以逗号或标签形式展示。
- 交互规则：筛选内容定位后，只展示命中任一选中定位的账号。
- 数据规则：取账号台账维护结果；支持多选筛选；默认不参与排序。

运行情况：

- 展示规则：展示正常、停更、待注销、失效等运行状态，并用状态标识区分正常与异常。
- 交互规则：点击统计卡或筛选项后，列表按运行情况过滤。
- 数据规则：取账号当前运行状态；支持精确筛选；不单独改变账号列表默认排序。

自主情况：

- 展示规则：展示自主运营、代运营或 `--`。
- 交互规则：顶部运营范围切换和筛选项均可影响该字段范围，二者同时存在时取交集。
- 数据规则：取账号运营归属；支持精确筛选；默认不参与排序。

运维负责人 / 联系方式：

- 展示规则：展示负责人名称和联系方式，缺失时展示 `--`。
- 交互规则：点击联系方式可复制；复制成功后给出轻提示。
- 数据规则：取账号台账维护信息；支持按负责人关键词模糊搜索；默认不参与排序。

粉丝数：

- 展示规则：展示最近一次可用粉丝数，支持万级缩写；缺失时展示「数据缺失」。
- 交互规则：表头可按粉丝数升序或降序排序。
- 数据规则：取账号最近一次采集或人工维护的粉丝数；采集失败不估算。

采集情况：

- 展示规则：展示自动采集、无法采集、未接入、采集异常等状态。
- 交互规则：筛选采集情况后列表刷新；采集异常账号在列表中展示异常提示。
- 数据规则：取采集状态结果或管理员维护状态；支持精确筛选；不单独改变账号列表默认排序。

是否纳入周报：

- 展示规则：展示纳入、未纳入、待确认等状态；缺失时展示「待确认」。
- 交互规则：具备修改权限的用户可在账号编辑中调整该状态；调整后只影响后续周报候选池。
- 数据规则：取账号台账中的周报纳入状态；支持筛选；默认不参与排序。

上次发稿时间 / 最新操作时间：

- 展示规则：按年月日时分秒展示，缺失时展示 `--`。
- 交互规则：表头支持排序；排序状态在切换页码时保持。
- 数据规则：上次发稿时间取账号最近一条内容发布时间；最新操作时间取账号台账最近一次维护时间；默认按最新操作时间倒序展示，无最新操作时间时按账号创建时间倒序展示。

操作入口：

- 展示规则：列表右侧固定展示「操作」入口，以下拉菜单承载编辑、采集、移出矩阵等动作。
- 交互规则：仅具备权限的角色展示可用操作；无权限时操作入口置灰或隐藏。
- 数据规则：操作可用性由账号状态、采集状态、周报纳入状态和当前用户权限共同决定。

### 3.5 账号操作

新增账号：

- 展示规则：表单展示账号名称、媒体类型、账号主页链接、属地、账号类型、内容定位、运维负责人和联系方式。
- 交互规则：必填项缺失时禁止保存并在字段下方提示；保存成功后关闭弹窗并刷新列表。
- 数据规则：保存前按平台账号标识或账号主页链接判重；新账号默认进入待核验状态，未确认前不进入周报候选池。

编辑账号：

- 展示规则：表单展示账号基础信息、组织归属、内容定位、运行情况、采集情况和是否纳入周报。
- 交互规则：修改后保存产生操作记录；关闭前有未保存修改时提示确认。
- 数据规则：编辑只影响后续统计和后续周报，既有报告引用的数据不被回溯改写。

启用 / 停用采集：

- 展示规则：操作菜单中展示采集相关动作及当前采集状态。
- 交互规则：启用后账号进入采集准备或采集中状态；停用后不再进入自动传播分析。
- 数据规则：仅具备可识别采集标识的账号允许启用采集；采集异常时保留异常原因和最近成功采集时间。

移出矩阵：

- 展示规则：危险操作需二次确认。
- 交互规则：确认后账号从矩阵列表移出，页面刷新统计数据。
- 数据规则：移出矩阵只影响后续统计，不删除既有内容、既有传播数据和既有报告引用记录。

批量删除 / 批量移出：

- 展示规则：列表支持勾选账号后执行批量操作；未选择账号时批量按钮置灰。
- 交互规则：点击后展示二次确认，说明本次影响账号数量；确认后刷新账号列表、统计卡和分页总数。
- 数据规则：仅影响被选中的矩阵账号；既有报告引用的数据不回溯改写；无权限账号不可被批量操作。

导出账号列表：

- 展示规则：列表上方展示导出入口。
- 交互规则：点击后导出当前筛选条件、关键词和排序状态下的账号列表；无结果时按钮置灰或提示暂无可导出数据。
- 数据规则：导出范围与页面当前筛选结果一致；导出内容需包含导出时的筛选条件摘要。

通知：

- 展示规则：具备通知权限时展示通知入口；未配置责任人或联系方式的账号不可发起通知。
- 交互规则：点击后选择通知对象并确认发送；发送成功后记录通知状态，失败时展示原因并允许重试。
- 数据规则：通知对象取账号台账中的运维负责人和联系方式；通知不改变账号采集状态、运行状态和周报纳入状态。

状态变更审核：

- 展示规则：「状态变更审核」tab 展示待处理、处理中、已通过、已驳回、已撤回等审核记录。
- 交互规则：点击审核记录可查看账号、申请状态、申请原因和处理结果；具备审核撤回权限的用户可撤回未完成处理的申请。
- 数据规则：审核记录取账号状态变更申请；撤回只影响未完成处理的审核记录，不改写已生效的账号状态。

### 3.6 本模块数据口径与异常

账号纳入口径：

- 未被移出矩阵。
- 账号运行情况属于有效状态，或处于需要展示的异常状态。
- 账号具备账号名称、媒体类型、账号主页链接等基础字段。
- 采集中账号必须具备平台账号标识或可识别的唯一采集标识。

判重规则：

- 优先按「平台 + 平台账号标识」判重。
- 缺失平台账号标识时，按「平台 + 账号主页链接」辅助判重。
- 重复账号不自动覆盖原数据，需运营确认后更新。

周报可用建议规则：

- 「重点展示」：纳入周报、采集中、周期内有发文，且传播量或互动量进入矩阵账号前 20%。
- 「常规展示」：纳入周报、采集中、周期内有发文，但未进入重点展示。
- 「不展示」：未纳入周报、未接入采集、采集异常、周期内无发文，或传播关键字段缺失且未人工确认。

异常规则：

- 账号采集异常时，列表展示异常状态，并提供异常原因、负责人和最近成功采集时间。
- 采集异常账号不进入有效传播统计；如已在既有报告中引用，既有报告保留原状态。
- 权限不足时，仅允许查看，不允许新增、编辑、移出矩阵或调整采集状态。

### 3.7 本模块交付验收

- 账号运行情况统计、内容定位统计和账号列表使用同一筛选范围。
- 支持按所属部门、媒体类型、运行情况、所属圈层、采集情况、属性、是否认证、内容定位、自主情况、关键词等条件筛选。
- 账号列表能展示采集状态、是否纳入周报、上次发稿时间和最新操作时间。
- 新增、编辑、移出矩阵、启用或停用采集均有明确状态反馈。
- 异常账号有状态提示，移出矩阵不删除既有内容和既有报告引用记录。

## 4. 模块二：矩阵传播分析

### 4.1 模块目标与边界

矩阵传播分析用于分析定向矩阵账号在指定周期内的内容发布与传播效果。该模块解决「矩阵账号发了什么」「传播效果如何」「哪些账号和稿件可进入周报候选」三个问题。

模块边界：

- 本模块负责周期筛选、传播指标、趋势图、账号传播详情和稿件传播明细。
- 本模块不维护账号基础信息，账号基础信息由「正能量矩阵」模块维护。
- 本模块不编辑报告正文，报告预览由「正能量热点报告 / 周报」模块承载。

### 4.2 筛选条件

入口规则：

- 展示规则：顶部导航点击「矩阵传播分析」进入本模块，并展示当前导航激活态。
- 数据规则：本模块入口仅在本期范围内展示，不新增其他未上线分析入口。

时间范围：

- 展示规则：提供「昨日、近7天、近30天、全年」快捷项，并展示自定义起止日期。
- 交互规则：点击快捷项后自动填充日期范围；修改自定义日期后取消快捷项激活态。
- 数据规则：取所选时间范围内矩阵账号发布内容和传播数据；自定义范围与平台、账号筛选取交集；日期按自然日聚合展示。

平台切换：

- 展示规则：以标签或分段控件展示全部、微信、微博、抖音、视频号、其他，并展示各平台内容数量。
- 交互规则：点击平台后，指标、趋势、账号传播详情和稿件传播明细同步刷新。
- 数据规则：平台取账号所属平台和内容发布平台；多平台内容按平台内容分别统计；默认展示全部平台。

账号检索：

- 展示规则：账号传播详情右侧展示账号名称搜索框。
- 交互规则：输入关键词后按账号名称模糊搜索，清空后恢复全部账号。
- 数据规则：检索范围为当前时间范围和平台筛选下的矩阵账号；默认按总互动量降序展示。

稿件检索：

- 展示规则：稿件传播明细右侧展示稿件标题搜索框，并提供账号下拉筛选。
- 交互规则：标题搜索和账号筛选可叠加；切换平台时保留标题关键词，账号筛选回到「全部」。
- 数据规则：标题采用模糊匹配，账号采用精确匹配，多条件取交集；默认按发布日期倒序展示。

数据导出：

- 展示规则：账号传播详情和稿件传播明细均展示导出入口。
- 交互规则：点击导出后下载当前筛选条件下的数据。
- 数据规则：导出范围与页面当前筛选结果一致；导出文件必须包含统计周期和筛选条件摘要。

### 4.3 核心指标

发稿数：

- 展示规则：指标卡展示周期内矩阵账号发布内容总数。
- 交互规则：随时间范围、平台和账号筛选变化实时刷新。
- 数据规则：统计当前筛选范围内符合正能量内容规则的内容数量；重复采集内容按内容去重。

总互动量：

- 展示规则：指标卡展示点赞、评论、转发、在看等互动量汇总。
- 交互规则：鼠标悬停可展示口径说明。
- 数据规则：取当前筛选范围内各平台可用互动字段合计；缺失字段不估算，并在明细中展示缺失状态。

点赞数 / 评论数 / 转发数：

- 展示规则：分别以指标卡展示当前周期内点赞、评论、转发汇总值。
- 交互规则：随筛选条件变化刷新；不可用字段展示「数据缺失」或 `0`，需与平台口径保持一致。
- 数据规则：取当前筛选范围内内容维度的对应互动字段；同一内容重复采集时取周期内最新可用值。

### 4.4 分析视图

传播力变化趋势：

- 展示规则：按日期展示全部平台及各平台的传播量趋势和互动量趋势，不合成为未定义的单一传播力分值；平台标签区分微信、微博、抖音、视频号、其他。
- 交互规则：点击平台标签可显示或隐藏对应平台趋势；鼠标悬停展示当日传播量、互动量和发稿数。
- 数据规则：按发布时间所在日期聚合；筛选条件与页面当前条件一致；日期按时间正序展示；传播量缺失的平台不绘制传播量趋势，但仍可展示互动量趋势和缺失说明。

账号传播详情：

- 展示规则：以表格展示账号名称、所属平台、发稿数、点赞数、阅读数、转发数、在看数、总互动量。
- 交互规则：平台 tab、账号搜索和表头排序均可影响表格结果；点击导出下载当前结果。
- 数据规则：取当前筛选范围内账号维度聚合结果；默认按总互动量降序展示；总互动量相同按发稿数降序展示。

稿件传播明细：

- 展示规则：以表格展示稿件标题、所属账号、所属平台、发布日期、点赞数、阅读数、转发数、在看数、总互动量。
- 交互规则：支持平台 tab、账号下拉、标题搜索、表头排序和导出。
- 数据规则：取当前筛选范围内内容维度明细；默认按发布日期倒序展示；用户点击指标表头后按对应指标排序。

### 4.5 本模块数据口径与异常

账号纳入口径：

- 账号未被移出矩阵。
- 运行情况为「正常」。
- 采集情况为「采集中」。
- 具备平台账号标识或可识别的唯一采集标识。

内容纳入口径：

- 发布时间落在当前统计周期内。
- 内容所属账号在统计周期内为有效矩阵账号。
- 内容命中正能量标签、正面情绪、稿池引用、热点事件关联或人工标记中的至少一种。
- 删除、下架、采集失败且无法核验的内容不进入统计。

传播量与互动量口径：

- 微博传播量取阅读量或视频播放量，互动量取转发、评论、点赞。
- 抖音传播量取播放量，互动量取点赞、评论、分享、收藏。
- 微信传播量取阅读量、看一看或在看，互动量取点赞、分享、留言。
- 视频号传播量取播放量，互动量取点赞、评论、转发、收藏。
- 客户端传播量取浏览量或阅读量，互动量取评论、点赞、分享。
- 如平台无传播量字段，应标记为「数据缺失」，不得用互动量或估算值替代。

去重规则：

- 同一内容跨平台发布时，按平台内容分别统计。
- 存在重复采集的内容，以平台内容标识去重。
- 缺失平台内容标识时，按「账号 + 标题 + 发布时间」辅助判重。

异常规则：

- 采集异常账号不纳入有效统计，并在明细或提示中说明原因。
- 传播量或互动量缺失时展示「数据缺失」，不做估算。
- 时间范围为空或结束时间早于开始时间时，禁止查询并提示用户修正。
- 当前筛选条件无数据时，指标卡展示 `0` 或「数据缺失」的真实状态，趋势图和明细表展示空状态说明。
- 当前筛选范围内所有传播量字段均缺失时，不展示传播量汇总值，仅展示缺失说明；互动量仍按可用字段统计。
- 无导出权限时导出按钮隐藏或置灰；筛选无结果时导出按钮置灰或提示暂无可导出数据。

### 4.6 本模块交付验收

- 支持自定义周期展示发布内容、传播量、互动量、账号表现和稿件明细。
- 指标卡、趋势图、账号传播详情和稿件传播明细使用同一统计周期和筛选条件。
- 账号传播详情和稿件传播明细可作为周报表 3 候选数据来源。
- 数据缺失、采集异常账号有明确提示，不参与虚假汇总。
- 导出数据与当前页面筛选结果一致。

## 5. 模块三：正能量热点报告 / 周报

### 5.1 模块目标与边界

正能量热点报告模块保留现有自动报告、人工报告的业务结构，并在自动报告类型中增加「周报」tab。周报能力不作为新的顶部一级菜单展示，而是作为现有报告模块的增强能力。

模块边界：

- 本模块负责日报列表、周报列表、周报设置页、候选数据挑选和预览。
- 本模块引用网讯舆情系统、新莓汇热点管理列表、新莓汇接收稿件列表和矩阵账号传播效果，不负责维护外部系统数据。
- 本模块不承诺按话题热门参与情况生成「正能量账号名单」。

### 5.2 报告列表入口

展示规则：

- 顶部导航点击「正能量热点报告」进入本模块，默认进入自动报告页。
- 一级 tab 展示自动报告、人工报告。
- 自动报告页内容区展示日报、周报类型切换；人工报告页不展示该类型切换。
- 筛选区展示发布时间起止日期和关键词搜索。
- 列表区以报告卡片展示报告名称、统计日期或统计周期。
- 鼠标悬停报告卡片时展示「预览」按钮，按钮不常驻占用列表空间。
- 分页区展示总数、页码和翻页控件。

交互规则：

- 点击「自动报告 / 人工报告」切换对应报告来源。
- 在自动报告页点击「日报」展示正能量热点报告卡片。
- 在自动报告页点击「周报」展示正能量周报卡片。
- 点击日报卡片默认打开日报预览。
- 点击周报卡片主体进入周报设置页。
- 点击卡片上的「预览」按钮仅打开报告预览弹窗，不触发进入设置页。
- 点击弹窗关闭按钮、遮罩或按 Esc 关闭预览弹窗。

数据规则：

- 取数规则：日报沿用现有正能量热点报告记录；周报取可配置周期记录或已有周报卡片数据。
- 数据筛选规则：按报告来源、报告类型、发布时间范围和关键词取交集筛选。
- 数据排序规则：默认按报告发布时间倒序展示。

### 5.3 周报设置页

展示规则：

- 顶部展示返回按钮、页面标题和预览按钮。
- 设置区展示统计周期的开始时间和结束时间，不在标题下重复展示周期文本。
- 候选数据区展示当前统计周期下与最终报告强相关的候选数据，按「热点事件、稿池投稿、矩阵账号」顺序组织。
- 候选数据以可挑选列表展示，每条数据支持保留、移除和恢复。

交互规则：

- 点击「返回」回到报告列表，并保留原报告类型 tab 状态。
- 修改统计周期后，页面状态提示同步展示当前周期。
- 点击候选数据类型 tab，可在热点事件、稿池投稿、矩阵账号之间切换。
- 点击候选数据的「移除」后，该数据不再进入最终报告正文、表格和预览；点击「恢复」可重新纳入。
- 点击「恢复全部」后，当前统计周期下所有候选数据重新纳入报告。
- 点击「预览」打开周报预览弹窗，预览内容需以报告正文版式展示总体情况、正能量稿件情感分布图、工作情况、热点事件表、稿池投稿表和矩阵账号传播表。

数据规则：

- 取数规则：取当前周报卡片对应统计周期下的热点事件、稿池投稿和矩阵账号传播效果三类候选数据。
- 数据筛选规则：按照统计周期以及运营保留/移除状态取交集。
- 数据排序规则：候选数据按热点事件、稿池投稿、矩阵账号的业务顺序分组展示；组内默认按传播量、篇次、互动量降序排序。
- 数据引用规则：预览和表格只引用处于保留状态的候选数据，被移除数据保留在当前操作态中用于恢复。

### 5.4 周报预览结构

展示规则：

- 标题展示「涉舟网络正能量数据简报」。
- 第一部分展示「一、总体情况」，包含网讯舆情系统统计的涉舟信息总量，并引用正能量态势感知首页的正能量稿件情感分布结果。
- 图 1 展示「正能量稿件情感分布图」。
- 第二部分展示「二、工作情况」，包含热点事件、稿池投稿和矩阵账号传播表现的整体描述。
- 表 1 展示涉舟正能量热点事件。
- 表 2 展示「新莓汇」正能量稿池投稿情况。
- 表 3 展示矩阵账号本周内容及传播效果。

交互规则：

- 候选数据被移除或恢复后，预览中的正文摘要、表格行和统计值同步更新。
- 预览弹窗只用于快速查看报告内容，不承诺新增独立编辑流程。
- 若表格无可用数据，保留表头并展示空状态说明。

数据规则：

- 取数规则：预览只引用当前统计周期内处于保留状态的候选数据；总体涉舟信息和情感分布按同一统计周期取数。
- 数据筛选规则：候选数据类型、统计周期和保留状态共同决定预览范围。
- 数据排序规则：表 1 按传播量降序排序；表 2 按篇次或报告展示优先级排序；表 3 按传播量、互动量、发文数排序。

### 5.5 周报时间范围

展示规则：

- 周报设置页展示起止时间，默认使用客户周报周期。
- 支持默认「上周三到本周三」以及手动调整具体小时和分钟。
- 起止时间在候选数据筛选和预览正文中保持一致。

交互规则：

- 修改时间范围后，候选数据和预览同步刷新。
- 起止时间为空或结束时间早于开始时间时，禁止预览，并提示用户修正。

数据规则：

- 取数规则：统计周期以运营在周报设置页选择的起止时间为准。
- 数据筛选规则：涉舟信息、正能量稿件情感分布、热点事件、稿池投稿和矩阵账号传播效果均按同一时间范围过滤。
- 数据排序规则：正文统计周期和系统查询条件必须使用同一时间范围。

### 5.6 周报统计口径

总体情况：

- 展示规则：展示当前周期内涉舟信息总量和总体描述。
- 交互规则：修改统计周期后同步更新；不可用指标展示「数据缺失」。
- 数据规则：
  - 取数规则：涉舟信息总量来自网讯舆情系统，监测方案为舟山，统计对象为主要新闻 APP、网站和社交平台中的涉舟信息。
  - 数据筛选规则：按周报设置页起止时间过滤；时间口径必须支持小时级，不能按自然日零点到零点替代。
  - 数据排序规则：总体情况为汇总指标，不涉及排序。

正能量稿件情感分布图：

- 展示规则：展示正能量态势感知首页「正能量稿件情感分布」图，包含赞扬、惊奇、喜悦、厌恶、愤怒、恐惧、悲伤等情感分类的数量和占比。
- 交互规则：预览弹窗中作为只读图表展示，不提供独立编辑；修改统计周期后随周报整体刷新。
- 数据规则：
  - 取数规则：来自正能量态势感知首页的正能量稿件情感分布统计，只统计当前周报周期内纳入正能量稿件范围的数据。
  - 数据筛选规则：按周报设置页起止时间过滤；无法识别或缺失情感结果的数据不强制归入任一情感分类。
  - 数据排序规则：按首页情感分布图既有分类顺序展示。

表 1 涉舟正能量热点事件：

- 展示规则：展示事件名称、属地、传播量、来源账号；传播量缺失时展示「数据缺失」。
- 交互规则：运营移除某一热点事件后，该事件不进入预览；恢复后重新纳入。
- 数据规则：
  - 取数规则：来源为新莓汇热点管理列表中的「全网涉浙 24 小时正能量热点」；统计对象为属地或检索条件命中舟山的正能量热点事件。
  - 数据筛选规则：先按舟山和日期范围筛选，再按周报设置页起止时间做小时级过滤；多条件取交集。
  - 数据排序规则：默认按阅读总量或传播量降序排序，传播量缺失的事件排在可用传播量事件之后。

表 2「新莓汇」正能量稿池投稿情况：

- 展示规则：展示投稿账号、篇次、热点选题稿件数量；无数据时保留表头并展示空状态说明。
- 交互规则：运营移除某一投稿主体后，该主体不进入预览；恢复后重新纳入。
- 数据规则：
  - 取数规则：来源为新莓汇工作台的接收稿件列表；统计对象为市新闻传媒中心、定海区融媒体中心、普陀区融媒体中心、岱山县融媒体中心、嵊泗县融媒体中心、嵊泗网信 6 个固定媒体单位的投稿。
  - 数据筛选规则：按推送时间过滤周报设置页起止时间；同一稿件重复出现时按稿件维度去重；热点选题稿件取处理状态为「市网信办已录用」的稿件。
  - 数据排序规则：默认按篇次降序排序，篇次相同时按固定媒体单位顺序展示。

表 3 矩阵账号本周内容及传播效果：

- 展示规则：展示属地、账号名称、平台、发文数、传播量、互动量和代表内容。
- 交互规则：运营移除某一矩阵账号后，该账号不进入预览；恢复后重新纳入。
- 数据规则：
  - 取数规则：来源为正能量矩阵账号及矩阵传播分析结果，只统计已纳入周报、可定向采集、且在统计周期内产生可核验内容数据的账号。
  - 数据筛选规则：按统计周期、是否纳入周报、运行情况、采集情况和运营保留状态取交集。
  - 数据排序规则：默认按传播量、互动量、发文数排序。

### 5.7 关键口径替换

原手动报告中的「正能量账号名单」隐含口径是从话题热门参与情况中识别账号。由于平台无法稳定识别所有话题参与账号，且不同平台对参与账号、互动账号、阅读来源等字段开放程度不一致，该口径不进入本次系统建设范围，也不在对外交付中承诺。

口径说明文案：

- 将账号名单调整为「正能量矩阵账号本周发布内容及传播效果」。
- 该表只统计客户确认纳入矩阵、具备定向采集条件、且在统计周期内产生可核验内容数据的账号。
- 对外展示口径：为保证周报数据可采集、可核验、可复用，系统不采用“按话题自动识别全部热门参与账号”的不稳定口径，调整为展示已纳入正能量矩阵的重点账号在本统计周期内的发布内容及传播效果。

### 5.8 本模块数据口径与异常

候选数据取数规则：

- 涉舟信息总量取网讯舆情系统中舟山监测方案的统计结果。
- 正能量稿件情感分布取正能量态势感知首页的「正能量稿件情感分布」统计结果。
- 热点事件取新莓汇热点管理列表中命中舟山并落在统计周期内的正能量热点事件。
- 稿池投稿取新莓汇工作台接收稿件列表中 6 个固定媒体单位的投稿统计，热点选题稿件取处理状态为「市网信办已录用」的数据。
- 矩阵账号传播效果取当前统计周期内符合表 3 口径的矩阵账号周期统计。

矩阵账号进入表 3 的条件：

- 「是否纳入周报」为是。
- 运行情况为正常，采集情况为采集中。
- 周期内至少有 1 条符合正能量内容规则的发布内容。
- 至少具备发文数和代表内容；传播量或互动量缺失时需要展示缺失状态。

异常规则：

- 候选数据全部被移除时，禁止预览并提示至少保留 1 条数据。
- 时间范围为空或结束时间早于开始时间时，禁止预览。
- 无矩阵账号数据时，表 3 展示空状态说明，不输出虚假账号。
- 存在关键字段缺失时，预览保留该数据并标记「数据缺失」，不使用估算值补齐。
- 网讯舆情系统或正能量稿件情感分布数据不可用时，总体情况和图表展示「数据缺失」，不使用候选数据反推。
- 周报预览失败时，展示失败原因，支持重新预览；失败任务不清空当前候选数据选择状态。

### 5.9 本模块交付验收

- 报告模块保留现有「自动报告 / 人工报告」结构。
- 自动报告页内容区新增「日报 / 周报」类型切换，周报不作为新的顶部一级菜单，也不与「自动报告 / 人工报告」平级。
- 报告卡片 hover 时展示「预览」按钮，点击只打开预览弹窗，不触发进入设置页。
- 点击周报卡片主体进入周报设置页，可返回报告列表。
- 周报设置页能展示统计周期、候选数据挑选和已选计数。
- 周报统计周期支持小时级起止时间，预览和候选数据筛选使用同一时间范围。
- 运营可移除或恢复候选数据，预览只引用保留下来的数据。
- 报告预览至少包含涉舟正能量热点事件、「新莓汇」正能量稿池投稿情况、矩阵账号本周内容及传播效果三张表。
- 图 1 必须明确为正能量态势感知首页「正能量稿件情感分布」口径，不得与网讯舆情系统涉舟信息总量混写。
- 表 1 至少展示事件、属地、传播量和来源账号。
- 表 2 至少展示投稿账号、篇次和热点选题稿件数量。
- 表 3 至少展示属地、账号名称、平台、发文数、传播量、互动量和代表内容。
