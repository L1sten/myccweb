# 公安移动端 H5 异常处置 产品需求文档

## 1. 产品概述

### 1.1 背景

公安新媒体矩阵在微信、微博、抖音等渠道持续发布内容，需要在移动端快速处理数据异常、巡检异常和评论异常。数据报告为线上已有模块，本 PRD 聚焦“主题填报”和“异常处理”相关移动端规则。

### 1.2 产品目标

- 将待处理异常按数据异常、巡检异常、评论异常分类展示，便于快速判断优先级。
- 支持异常忽略、异常处置、处置记录回看，形成移动端轻量闭环。
- 支持从异常列表进入详情页，查看巡检命中问题或评论原文快照。
- 支持用户通过微信通知卡片进入 H5 时免密登录，减少移动端处理阻力。

### 1.3 用户角色

- **省级公安宣传管理员**：查看全省矩阵账号异常，统筹异常处置。
- **市县级矩阵账号负责人**：处理本账号相关异常。
- **内容巡检人员**：关注表述、错别字、空格、时政重点词等巡检异常，并给出处置意见。

## 3. 登录模块优化

### 3.1 微信通知卡片免密登录

- **展示规则**
  - 用户通过微信通知卡片打开 H5 时，系统优先尝试免密登录。
  - 免密登录成功后直接进入通知卡片对应的业务页面，例如异常详情页或异常处理列表页。
  - 免密登录过程中不展示账号密码输入页，不要求用户重复输入密码。
  - 免密登录失败、通知凭证过期、用户绑定关系缺失或账号无权限时，进入常规登录或验证码校验流程。
- **交互规则**
  - 通知发送前已按手机号匹配微信接收人，系统已掌握接收人、手机号、微信用户三者绑定关系；免密登录以该绑定关系作为校验前提。
  - 微信通知卡片链接携带一次性登录凭证和业务跳转参数，不在前端链接中明文携带手机号、微信用户标识或用户敏感信息。
  - 用户点击通知卡片后，H5 调用免密登录接口，后端根据通知发送记录校验接收人、手机号、微信用户和系统用户的绑定关系。
  - 校验通过后建立 H5 登录态，并按通知卡片携带的业务跳转参数进入目标页面。
  - 当同一手机号关联多个可用账号或多单位权限时，系统应按既有权限策略进入默认身份；若无法自动判定，则展示身份选择流程。
  - 同一个一次性登录凭证仅允许使用一次，重复打开时若当前登录态仍有效则直接进入目标页面，否则进入常规登录流程。
- **数据规则**
  - **取数规则**：读取微信通知发送记录、手机号与微信用户绑定关系、手机号与系统用户绑定关系、用户角色权限和目标业务参数。
  - **数据筛选规则**：仅允许通知实际接收人对应的绑定用户完成免密登录；不得通过修改跳转参数切换用户身份或越权访问业务数据。
  - **数据排序规则**：登录链路本身无排序规则；涉及多账号选择时按既有账号优先级或最近使用账号展示。

### 3.2 安全与审计

- **展示规则**
  - 免密登录成功后用户无感进入业务页面；失败时提示完成身份验证，不暴露失败校验细节。
  - 登录态失效后再次访问业务页面，应回到常规登录或验证码校验流程。
- **交互规则**
  - 免密登录凭证由后端生成并签名，绑定通知 ID、接收手机号、微信用户标识、目标业务参数和有效期。
  - 凭证有效期应较短，过期、重复使用、签名异常、绑定关系不一致时均不允许免密登录。
  - 前端仅负责携带凭证请求登录接口，不在本地持久化手机号、微信用户标识等敏感信息。
- **数据规则**
  - **取数规则**：记录免密登录审计日志，包括通知 ID、脱敏手机号、系统用户 ID、微信用户标识摘要、登录结果、失败原因、登录时间、客户端信息和目标页面。
  - **数据筛选规则**：审计日志按权限可查，普通业务用户不可查看他人登录审计记录。
  - **数据排序规则**：审计记录默认按登录时间倒序排列。

## 4. 异常处理

### 4.1 底部导航

- **展示规则**
  - 底部固定展示“数据报告”“主题填报”和“异常处理”三个入口。
  - 当前入口使用蓝色图标和文字高亮。
- **交互规则**
  - 点击“数据报告”进入线上已有数据报告模块；本 PRD 不展开该模块规则。
  - 点击“主题填报”无页面跳转和状态变化，用于保留线上已有模块入口位置。
  - 点击“异常处理”切换至异常列表，并保持最近一次选择的异常分类。
- **数据规则**
  - **取数规则**：导航不直接取数，切换后由对应页面读取数据。
  - **数据筛选规则**：切换到异常处理时保留当前异常分类状态；主题填报入口在本原型中不参与筛选。
  - **数据排序规则**：导航本身无排序规则。

### 4.2 日期筛选

- **展示规则**
  - 异常处理页在异常分类下方展示日期筛选框。
  - 默认不选择日期，日期筛选框展示“请选择日期”。
  - 未选择日期时不展示前一天、后一天按钮，避免出现不可用控件。
  - 选择日期后展示当前选中日期，并提供前一天、后一天快捷切换入口。
  - 选择日期后，日期筛选框内展示清空入口，用于清空当前日期筛选。
  - 点击日期筛选框打开底部日历弹层，弹层展示当前月份、星期行、日期网格、取消和确定按钮。
  - 日历弹层中当前选中日期高亮展示，当天日期展示圆点标记。
  - 可选日期最近截止到当天。
- **交互规则**
  - 点击日期区域打开底部日历弹层，选择日期后需点击“确定”才刷新当前异常分类列表。
  - 不选择日期时展示全部未处理异常。
  - 清空日期筛选框后恢复展示全部未处理异常。
  - 点击前一天或后一天后更新日期，并保持当前异常分类不变。
  - 当已选日期为当天时，后一天入口不可继续切换；日历弹层中的未来日期置灰且不可选。
  - 点击日历弹层遮罩或“取消”关闭弹层，不改变当前筛选条件。
  - 切换日期后列表回到顶部。
- **数据规则**
  - **取数规则**：读取当前用户权限范围内所选日期产生或发现的异常数据。
  - **数据筛选规则**：未选择日期时不按日期过滤，仅展示全部未处理异常；选择日期后，日期筛选作用于数据异常、巡检异常和评论异常三个分类，并与异常分类、平台、账号权限等筛选条件共同生效。
  - **数据排序规则**：默认按异常发现时间倒序排列；同一日期内同样按异常发现时间倒序排列。

### 4.3 异常分类切换

- **展示规则**
  - 顶部展示“数据异常”“巡检异常”“评论异常”三个分段选项。
  - 默认选中“数据异常”。
  - 日期筛选下方展示当前分类的数据总数，格式为“共 X 项数据”，不展示额外状态标签。
- **交互规则**
  - 点击分类后切换选中态，并刷新列表卡片内容。
  - 切换分类时列表回到顶部。
- **数据规则**
  - **取数规则**：读取当前用户权限范围内状态为待处理的异常。
  - **数据筛选规则**：按异常类型精确筛选；仅展示待处理状态数据。
  - **数据排序规则**：按异常发现时间倒序排列。

### 4.4 异常卡片通用规则

- **展示规则**
  - 每张卡片展示账号名称、平台标签、异常摘要、发现时间、忽略按钮和处置按钮。
  - 平台标签包含微信、微博、抖音，不同平台使用不同底色。
  - 数据异常、巡检异常、评论异常使用统一卡片骨架，账号、平台标签和待处理状态位置保持一致。
  - 异常摘要区域使用浅灰背景，左侧展示告警图标。
  - 异常摘要区域右侧展示文本入口：数据异常和巡检异常为“查看详情”，评论异常为“查看原文”。
  - 时间行右侧统一展示异常原因标签；巡检异常展示巡检原因，评论异常展示 AI 识别原因，数据异常不展示原因标签。
- **交互规则**
  - 点击“忽略”后将该异常标记为无需处理，并从当前列表移除。
  - 点击“处置”打开处置弹窗。
  - 点击数据异常卡片或“查看详情”进入数据异常详情页；点击巡检异常摘要或“查看详情”进入表述异常详情；评论异常卡片或“查看原文”按评论异常规则跳转原文地址。
  - 点击“忽略”或“处置”时不触发卡片详情跳转。
- **数据规则**
  - **取数规则**：从异常中心读取账号、平台、异常摘要、异常类型、发现时间和处置状态。
  - **数据筛选规则**：受异常分类、用户权限、处置状态共同约束。
  - **数据排序规则**：默认按发现时间倒序排列；同一时间按异常等级从高到低排列。

### 4.5 数据异常

- **展示规则**
  - 摘要内容展示异常指标和波动比例，例如“阅读数 -859.88%”。
  - 负向和正向异常均展示为待处理项。
- **交互规则**
  - 点击数据异常卡片进入数据异常详情页，查看异常指标波动和关联内容明细。
  - 处置成功后该卡片从数据异常列表移除。
- **数据规则**
  - **取数规则**：读取矩阵账号数据监测结果中超过阈值的指标波动。
  - **数据筛选规则**：仅展示达到异常阈值且未处理的指标项；平台、账号权限按当前用户范围过滤。
  - **数据排序规则**：按异常发现时间倒序排列；同一时间按波动绝对值从大到小排列。

### 4.6 巡检异常

- **展示规则**
  - 摘要内容展示稿件标题。
  - 异常类型展示在时间行右侧，例如“表述异常”。
  - 长标题单行省略。
- **交互规则**
  - 点击摘要进入表述异常详情页，查看命中问题、修改建议和上下文。
  - 可直接忽略或处置。
- **数据规则**
  - **取数规则**：读取内容巡检系统识别出的待处理异常稿件。
  - **数据筛选规则**：仅展示巡检结论为异常且尚未处理的数据；可按异常类型归类。
  - **数据排序规则**：按巡检完成时间倒序排列。

### 4.7 评论异常

- **展示规则**
  - 评论异常参考 Web 端评论监测表格，在移动端转为卡片列表。
  - 评论异常不展示渠道筛选和评论内容搜索框。
  - 评论异常列表计数与其他异常分类一致，展示“共 X 项数据”。
  - 每张评论卡片沿用异常卡片通用结构，顶部展示账号、平台标签和“待处理”状态。
  - 评论正文展示在异常摘要区域，摘要区域右侧展示“查看原文”，评论时间展示在时间行左侧，AI 识别结果展示在时间行右侧。
  - AI 识别结果包含“质疑执法”“质疑处理结果”等风险标签。
  - 异常处理列表仅展示“未处理”评论；已处理评论不进入待处理列表。
- **交互规则**
  - 点击评论卡片跳转原文地址。
  - 评论异常操作按钮与数据异常、巡检异常保持一致，展示“忽略”和“处置”。
  - 点击“忽略”或“处置”时不触发卡片跳转。
- **数据规则**
  - **取数规则**：读取评论监测模型识别出的疑似负面或风险评论，并限制为未处理状态。
  - **数据筛选规则**：按全局日期、账号权限、AI 识别结果和未处理状态过滤。
  - **数据排序规则**：移动端与 Web 端评论监测列表保持一致排序，默认按评论发现时间倒序排列。

## 5. 数据异常详情

### 5.1 指标波动概览

- **展示规则**
  - 顶部展示返回按钮和标题“数据异常详情”。
  - 内容区展示账号名称、平台、发现时间、异常指标名称、波动文案和波动比例。
  - 指标名称示例为“点赞数”，波动比例示例为“+616.00%”。
  - 正向和负向波动均作为异常展示，波动比例使用醒目样式。
- **交互规则**
  - 点击返回回到数据异常列表，并保留原异常分类和日期筛选状态。
  - 指标波动概览仅用于查看，不支持编辑。
- **数据规则**
  - **取数规则**：读取当前数据异常记录的账号、平台、发现时间、异常指标、波动方向和波动比例。
  - **数据筛选规则**：仅读取当前被点击的数据异常记录，且必须符合用户权限范围。
  - **数据排序规则**：概览区域无排序规则。

### 5.2 关联内容明细

- **展示规则**
  - 关联内容区域展示命中该指标异常的内容列表。
  - 每条内容展示正文标题或摘要，以及对应指标值，例如“点赞数：100”。
  - 内容标题使用可点击样式，指标值在同一行右侧展示。
- **交互规则**
  - 点击内容标题进入对应原文或内容详情；原型中以提示反馈代替真实跳转。
  - 关联内容列表不展示处置按钮，处置仍在异常列表卡片或处置弹窗中完成。
- **数据规则**
  - **取数规则**：读取当前异常指标下贡献波动的内容明细及对应指标值。
  - **数据筛选规则**：仅展示当前异常记录关联的内容；内容范围受账号权限、平台权限和异常日期共同约束。
  - **数据排序规则**：默认按指标值倒序排列；指标值相同按内容发布时间倒序排列。

## 6. 异常处置

### 6.1 处置弹窗

- **展示规则**
  - 弹窗居中展示，背景增加半透明遮罩。
  - 标题为“处置”，包含关闭按钮。
  - 处置结果提供三个单选项：已核查并处理该异常、已核实并反馈上级部门、已核实无此异常。
  - 文本框提示为“请输入处置结果”，右下角展示字数统计。
  - 下方展示历史处置记录，包含操作单位、账号、处置结论和处置时间。
  - 底部展示“取消”和“确定”两个按钮。
- **交互规则**
  - 点击关闭、取消或遮罩关闭弹窗，不保存操作。
  - 必须选择处置结果后才允许确定。
  - 文本框最多输入 200 字，超出后阻止继续输入。
  - 点击确定后保存处置记录，关闭弹窗，并将当前异常标记为已处置。
- **数据规则**
  - **取数规则**：打开弹窗时读取当前异常的历史处置记录。
  - **数据筛选规则**：仅展示当前异常对应的处置记录。
  - **数据排序规则**：历史处置记录按处置时间倒序排列。

### 6.2 状态流转

- **展示规则**
  - 待处理状态显示“忽略”和“处置”按钮。
  - 已处置或无需处理状态不在待处理列表展示。
- **交互规则**
  - 忽略：待处理变为无需处理。
  - 确定处置：待处理变为已处置。
  - 状态变更后刷新列表计数。
- **数据规则**
  - **取数规则**：状态来自异常中心的处置状态。
  - **数据筛选规则**：待处理列表仅展示状态为待处理的数据。
  - **数据排序规则**：状态变化不改变历史记录排序。

## 7. 表述异常详情

### 7.1 详情头部

- **展示规则**
  - 顶部展示返回按钮和标题“表述异常详情”。
  - 内容区展示账号名称、平台、发现时间和红色异常提示卡。
- **交互规则**
  - 点击返回回到巡检异常列表。
  - 异常提示卡仅用于说明，不可点击。
- **数据规则**
  - **取数规则**：读取所选巡检异常的账号、平台、发现时间和异常结论。
  - **数据筛选规则**：仅读取当前异常记录。
  - **数据排序规则**：详情头部无排序。

### 7.2 问题分类与建议

- **展示规则**
  - 展示筛选标签：全部、时政重点词、错别字、空格检查。
  - 问题卡展示问题类型、原文片段、建议替换文本和说明。
  - 原文问题词使用红色强调，建议文本使用绿色强调。
- **交互规则**
  - 点击标签后筛选问题卡。
  - 点击返回图标回到上一页。
- **数据规则**
  - **取数规则**：读取内容巡检结果中的问题明细和修改建议。
  - **数据筛选规则**：按问题分类筛选；“全部”展示所有问题。
  - **数据排序规则**：按问题在正文中出现的顺序排列。

## 8. 评论异常详情

### 8.1 异常概览

- **展示规则**
  - 顶部展示返回按钮和标题“评论异常详情”。
  - 内容区展示账号名称、平台、发现时间和橙色异常提示卡。
  - 异常提示文案为“检测到评论区存在疑似负面内容”。
- **交互规则**
  - 点击返回回到评论异常列表。
  - 异常提示卡不可点击。
- **数据规则**
  - **取数规则**：读取所选评论异常的账号、平台、发现时间和异常结论。
  - **数据筛选规则**：仅读取当前评论异常记录。
  - **数据排序规则**：详情概览无排序。

### 8.2 原文快照

- **展示规则**
  - 展示“原文快照”标题。
  - 快照区域展示微博正文样式，包括账号信息、正文、图片/视频区域、互动数据。
- **交互规则**
  - 点击“微博内打开”尝试跳转原文；原型中以提示反馈代替真实跳转。
  - 快照内容仅用于查看，不支持编辑。
- **数据规则**
  - **取数规则**：读取评论异常关联的原文快照或采集截图。
  - **数据筛选规则**：仅展示当前异常关联原文。
  - **数据排序规则**：原文快照无排序。

## 9. 全局规则

### 9.1 空状态与加载

- **展示规则**
  - 列表为空时展示“没有更多了”或空状态提示。
  - 原型中默认展示演示数据。
- **交互规则**
  - 筛选后无数据时保留当前筛选状态。
- **数据规则**
  - **取数规则**：根据当前页面和筛选条件读取数据。
  - **数据筛选规则**：多筛选条件取交集。
  - **数据排序规则**：空状态不展示排序结果。

### 9.2 权限范围

- **展示规则**
  - 用户仅看到自己权限范围内的异常。
- **交互规则**
  - 无权限数据不可见，不展示禁用项。
- **数据规则**
  - **取数规则**：按登录用户所属单位、角色和账号负责人关系读取数据。
  - **数据筛选规则**：权限过滤优先于页面筛选。
  - **数据排序规则**：权限过滤后再执行页面排序。

### 9.3 移动端适配

- **展示规则**
  - 页面以手机 H5 为主要使用场景，内容纵向滚动。
  - 底部导航固定在屏幕底部。
- **交互规则**
  - 主操作按钮保持可触达。
  - 详情页返回按钮固定在顶部导航内。
- **数据规则**
  - **取数规则**：移动端与桌面端使用一致业务数据口径。
  - **数据筛选规则**：移动端不减少业务筛选条件。
  - **数据排序规则**：移动端与桌面端保持一致排序口径。
