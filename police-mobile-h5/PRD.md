# 公安移动端 H5 异常处置与数据报告 产品需求文档

## 1. 产品概述

### 1.1 背景

公安新媒体矩阵在微信、微博、抖音等渠道持续发布内容，需要在移动端快速查看数据报告，并及时处理数据异常、巡检异常和评论异常。Figma 原型展示了一个面向手机访问的 H5 工具，围绕“数据报告”“主题填报”和“异常处理”三个底部入口组织工作。

### 1.2 产品目标

- 为公安宣传和新媒体运营人员提供移动端数据报告查看入口。
- 将待处理异常按数据异常、巡检异常、评论异常分类展示，便于快速判断优先级。
- 支持异常忽略、异常处置、处置记录回看，形成移动端轻量闭环。
- 支持从异常列表进入详情页，查看巡检命中问题或评论原文快照。

### 1.3 用户角色

- **省级公安宣传管理员**：查看全省矩阵账号报告和异常，统筹异常处置。
- **市县级矩阵账号负责人**：处理本账号相关异常，查看本账号报告。
- **内容巡检人员**：关注表述、错别字、空格、时政重点词等巡检异常，并给出处置意见。

## 3. 数据报告

### 3.1 周期筛选

- **展示规则**
  - 页面顶部展示分段控件，包含“日报”“周报”“月报”三个选项。
  - 默认选中“日报”。
  - 右侧展示日期选择入口。
- **交互规则**
  - 点击周期选项后切换高亮状态，并刷新报告列表。
  - 点击日期入口后进入日期选择流程；原型中以提示反馈代替真实选择器。
- **数据规则**
  - **取数规则**：读取当前用户权限范围内已生成的数据报告。
  - **数据筛选规则**：按日报、周报、月报筛选报告类型；日期筛选限定报告所属日期或周期。
  - **数据排序规则**：默认按报告日期倒序排列，最新报告排在最前。

### 3.2 报告卡片

- **展示规则**
  - 列表以卡片形式展示报告名称、报告日期、未读标识、预览入口和下载入口。
  - 报告名称示例为“新媒体健康度分析日报”。
  - 未读报告在卡片右上角展示红色角标。
- **交互规则**
  - 点击“预览”进入报告预览页。
  - 点击“下载”触发文件下载反馈。
  - 已读报告不展示未读角标。
- **数据规则**
  - **取数规则**：从报告库读取当前用户可见的报告记录。
  - **数据筛选规则**：报告列表受周期筛选、日期筛选和用户权限范围共同约束。
  - **数据排序规则**：按报告生成时间倒序排列；同一天多份报告按更新时间倒序排列。

### 3.3 报告预览页

- **展示规则**
  - 预览页以列表展示报告条目标题及核心指标，例如阅读数、在看数、点赞数、转发数。
  - 标题使用蓝色链接样式，指标在右侧展示。
- **交互规则**
  - 点击返回回到数据报告列表。
  - 点击报告条目可进入原文或详情；原型中以提示反馈代替外链跳转。
- **数据规则**
  - **取数规则**：读取所选报告下的明细内容和指标数据。
  - **数据筛选规则**：仅展示当前报告范围内的内容条目。
  - **数据排序规则**：默认按报告中的推荐顺序展示；无推荐顺序时按阅读数倒序展示。

## 4. 异常处理

### 4.1 底部导航

- **展示规则**
  - 底部固定展示“数据报告”“主题填报”和“异常处理”三个入口。
  - 当前入口使用蓝色图标和文字高亮。
- **交互规则**
  - 点击“数据报告”切换至报告列表。
  - 点击“主题填报”无页面跳转和状态变化，用于保留线上已有模块入口位置。
  - 点击“异常处理”切换至异常列表，并保持最近一次选择的异常分类。
- **数据规则**
  - **取数规则**：导航不直接取数，切换后由对应页面读取数据。
  - **数据筛选规则**：切换到异常处理时保留当前异常分类状态；主题填报入口在本原型中不参与筛选。
  - **数据排序规则**：导航本身无排序规则。

### 4.2 日期筛选

- **展示规则**
  - 异常处理页在异常分类下方展示日期筛选框。
  - 默认不选择日期，日期筛选框展示“全部日期”。
  - 未选择日期时不展示前一天、后一天按钮，避免出现不可用控件。
  - 选择日期后展示当前选中日期，并提供前一天、后一天快捷切换入口。
  - 已选择日期时展示“全部”入口，用于清空日期筛选。
  - 日期选择入口使用系统日期选择能力。
  - 可选日期最近截止到当天。
- **交互规则**
  - 点击日期区域打开日期选择器，选择后刷新当前异常分类列表。
  - 不选择日期时展示全部未处理异常。
  - 点击“全部”清空日期筛选，并恢复展示全部未处理异常。
  - 点击前一天或后一天后更新日期，并保持当前异常分类不变。
  - 当已选日期为当天时，后一天入口不可继续切换。
  - 切换日期后列表回到顶部。
- **数据规则**
  - **取数规则**：读取当前用户权限范围内所选日期产生或发现的异常数据。
  - **数据筛选规则**：未选择日期时不按日期过滤，仅展示全部未处理异常；选择日期后，日期筛选作用于数据异常、巡检异常和评论异常三个分类，并与异常分类、平台、账号权限等筛选条件共同生效。
  - **数据排序规则**：默认按异常发现时间倒序排列；同一日期内同样按异常发现时间倒序排列。

### 4.3 异常分类切换

- **展示规则**
  - 顶部展示“数据异常”“巡检异常”“评论异常”三个分段选项。
  - 默认选中“数据异常”。
  - 日期筛选下方展示当前分类的数据总数，格式为“共 X 项数据”，不展示额外状态标签。
- **交互规则**
  - 点击分类后切换选中态，并刷新列表卡片内容。
  - 切换分类时列表回到顶部。
- **数据规则**
  - **取数规则**：读取当前用户权限范围内状态为待处理的异常。
  - **数据筛选规则**：按异常类型精确筛选；仅展示待处理状态数据。
  - **数据排序规则**：按异常发现时间倒序排列。

### 4.4 异常卡片通用规则

- **展示规则**
  - 每张卡片展示账号名称、平台标签、异常摘要、发现时间、忽略按钮和处置按钮。
  - 平台标签包含微信、微博、抖音，不同平台使用不同底色。
  - 异常摘要区域使用浅灰背景，左侧展示告警图标。
- **交互规则**
  - 点击“忽略”后将该异常标记为无需处理，并从当前列表移除。
  - 点击“处置”打开处置弹窗。
  - 点击异常摘要进入详情页；评论异常进入评论异常详情，巡检异常进入表述异常详情。
- **数据规则**
  - **取数规则**：从异常中心读取账号、平台、异常摘要、异常类型、发现时间和处置状态。
  - **数据筛选规则**：受异常分类、用户权限、处置状态共同约束。
  - **数据排序规则**：默认按发现时间倒序排列；同一时间按异常等级从高到低排列。

### 4.5 数据异常

- **展示规则**
  - 摘要内容展示异常指标和波动比例，例如“阅读数 -859.88%”。
  - 负向和正向异常均展示为待处理项。
- **交互规则**
  - 点击摘要进入数据异常关联报告或指标详情；原型中以提示反馈代替详情页。
  - 处置成功后该卡片从数据异常列表移除。
- **数据规则**
  - **取数规则**：读取矩阵账号数据监测结果中超过阈值的指标波动。
  - **数据筛选规则**：仅展示达到异常阈值且未处理的指标项；平台、账号权限按当前用户范围过滤。
  - **数据排序规则**：按异常发现时间倒序排列；同一时间按波动绝对值从大到小排列。

### 4.6 巡检异常

- **展示规则**
  - 摘要内容展示稿件标题，下一行展示异常类型，例如“表述异常”。
  - 长标题单行省略。
- **交互规则**
  - 点击摘要进入表述异常详情页，查看命中问题、修改建议和上下文。
  - 可直接忽略或处置。
- **数据规则**
  - **取数规则**：读取内容巡检系统识别出的待处理异常稿件。
  - **数据筛选规则**：仅展示巡检结论为异常且尚未处理的数据；可按异常类型归类。
  - **数据排序规则**：按巡检完成时间倒序排列。

### 4.7 评论异常

- **展示规则**
  - 评论异常参考 Web 端评论监测表格，在移动端转为卡片列表。
  - 评论异常不展示渠道筛选和评论内容搜索框。
  - 评论异常列表计数与其他异常分类一致，展示“共 X 项数据”。
  - 每张卡片展示序号、平台、账号、正文内容、评论时间、AI 识别结果、处理状态和操作按钮。
  - AI 识别结果包含“质疑执法”“质疑处理结果”等风险标签。
  - 异常处理列表仅展示“未处理”评论；已处理评论不进入待处理列表。
- **交互规则**
  - 点击评论卡片跳转原文地址。
  - 评论异常操作按钮与数据异常、巡检异常保持一致，展示“忽略”和“处置”。
  - 点击“忽略”或“处置”时不触发卡片跳转。
- **数据规则**
  - **取数规则**：读取评论监测模型识别出的疑似负面或风险评论，并限制为未处理状态。
  - **数据筛选规则**：按全局日期、账号权限、AI 识别结果和未处理状态过滤。
  - **数据排序规则**：移动端与 Web 端评论监测列表保持一致排序，默认按评论发现时间倒序排列。

## 5. 异常处置

### 5.1 处置弹窗

- **展示规则**
  - 弹窗居中展示，背景增加半透明遮罩。
  - 标题为“处置”，包含关闭按钮。
  - 处置结果提供三个单选项：已核查并处理该异常、已核实并反馈上级部门、已核实无此异常。
  - 文本框提示为“请输入处置结果”，右下角展示字数统计。
  - 下方展示历史处置记录，包含操作单位、账号、处置结论和处置时间。
  - 底部展示“取消”和“确定”两个按钮。
- **交互规则**
  - 点击关闭、取消或遮罩关闭弹窗，不保存操作。
  - 必须选择处置结果后才允许确定。
  - 文本框最多输入 200 字，超出后阻止继续输入。
  - 点击确定后保存处置记录，关闭弹窗，并将当前异常标记为已处置。
- **数据规则**
  - **取数规则**：打开弹窗时读取当前异常的历史处置记录。
  - **数据筛选规则**：仅展示当前异常对应的处置记录。
  - **数据排序规则**：历史处置记录按处置时间倒序排列。

### 5.2 状态流转

- **展示规则**
  - 待处理状态显示“忽略”和“处置”按钮。
  - 已处置或无需处理状态不在待处理列表展示。
- **交互规则**
  - 忽略：待处理变为无需处理。
  - 确定处置：待处理变为已处置。
  - 状态变更后刷新列表计数。
- **数据规则**
  - **取数规则**：状态来自异常中心的处置状态。
  - **数据筛选规则**：待处理列表仅展示状态为待处理的数据。
  - **数据排序规则**：状态变化不改变历史记录排序。

## 6. 表述异常详情

### 6.1 详情头部

- **展示规则**
  - 顶部展示返回按钮和标题“表述异常详情”。
  - 内容区展示账号名称、平台、发现时间和红色异常提示卡。
- **交互规则**
  - 点击返回回到巡检异常列表。
  - 异常提示卡仅用于说明，不可点击。
- **数据规则**
  - **取数规则**：读取所选巡检异常的账号、平台、发现时间和异常结论。
  - **数据筛选规则**：仅读取当前异常记录。
  - **数据排序规则**：详情头部无排序。

### 6.2 问题分类与建议

- **展示规则**
  - 展示筛选标签：全部、时政重点词、错别字、空格检查。
  - 问题卡展示问题类型、原文片段、建议替换文本和说明。
  - 原文问题词使用红色强调，建议文本使用绿色强调。
- **交互规则**
  - 点击标签后筛选问题卡。
  - 点击返回图标回到上一页。
- **数据规则**
  - **取数规则**：读取内容巡检结果中的问题明细和修改建议。
  - **数据筛选规则**：按问题分类筛选；“全部”展示所有问题。
  - **数据排序规则**：按问题在正文中出现的顺序排列。

## 7. 评论异常详情

### 7.1 异常概览

- **展示规则**
  - 顶部展示返回按钮和标题“评论异常详情”。
  - 内容区展示账号名称、平台、发现时间和橙色异常提示卡。
  - 异常提示文案为“检测到评论区存在疑似负面内容”。
- **交互规则**
  - 点击返回回到评论异常列表。
  - 异常提示卡不可点击。
- **数据规则**
  - **取数规则**：读取所选评论异常的账号、平台、发现时间和异常结论。
  - **数据筛选规则**：仅读取当前评论异常记录。
  - **数据排序规则**：详情概览无排序。

### 7.2 原文快照

- **展示规则**
  - 展示“原文快照”标题。
  - 快照区域展示微博正文样式，包括账号信息、正文、图片/视频区域、互动数据。
- **交互规则**
  - 点击“微博内打开”尝试跳转原文；原型中以提示反馈代替真实跳转。
  - 快照内容仅用于查看，不支持编辑。
- **数据规则**
  - **取数规则**：读取评论异常关联的原文快照或采集截图。
  - **数据筛选规则**：仅展示当前异常关联原文。
  - **数据排序规则**：原文快照无排序。

## 8. 全局规则

### 8.1 空状态与加载

- **展示规则**
  - 列表为空时展示“没有更多了”或空状态提示。
  - 原型中默认展示演示数据。
- **交互规则**
  - 筛选后无数据时保留当前筛选状态。
- **数据规则**
  - **取数规则**：根据当前页面和筛选条件读取数据。
  - **数据筛选规则**：多筛选条件取交集。
  - **数据排序规则**：空状态不展示排序结果。

### 8.2 权限范围

- **展示规则**
  - 用户仅看到自己权限范围内的报告和异常。
- **交互规则**
  - 无权限数据不可见，不展示禁用项。
- **数据规则**
  - **取数规则**：按登录用户所属单位、角色和账号负责人关系读取数据。
  - **数据筛选规则**：权限过滤优先于页面筛选。
  - **数据排序规则**：权限过滤后再执行页面排序。

### 8.3 移动端适配

- **展示规则**
  - 页面以手机 H5 为主要使用场景，内容纵向滚动。
  - 底部导航固定在屏幕底部。
- **交互规则**
  - 主操作按钮保持可触达。
  - 详情页返回按钮固定在顶部导航内。
- **数据规则**
  - **取数规则**：移动端与桌面端使用一致业务数据口径。
  - **数据筛选规则**：移动端不减少业务筛选条件。
  - **数据排序规则**：移动端与桌面端保持一致排序口径。
