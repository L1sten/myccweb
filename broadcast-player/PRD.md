# 传播投手 — 发布传播需求 PRD

## 产品概述

### 背景

会展行业传播需求分散、对接效率低，客户需要一个标准化的需求提交入口，将传播需求（微博热搜、抖音热推、稿件推送、央媒传播、定制传播）统一收集，由运营团队快速匹配方案并执行。

### 目标

- 为客户提供 3 步极简的需求提交流程
- 结构化采集需求信息，减少沟通成本
- 提交后自动通知运营团队，24 小时内完成对接

### 用户角色

- **客户**：会展主办方或参展企业，需要传播推广服务
- **运营管理员**：接收需求、匹配方案、协调执行（本版本不含管理后台）

## 功能模块

### 一、发布传播需求（3 步表单）

#### Step 1：基本信息

**字段列表：**

- **任务名称**（必填）
  - 展示规则：单行文本输入框，placeholder「请输入任务名称」
  - 数据规则：最大长度 50 字符，超出截断
  - 校验规则：不可为空

- **任务起止时间**（必填）
  - 展示规则：两个日期选择器，中间用「至」连接
  - 数据规则：开始日期 ≤ 结束日期，不可选过去日期
  - 校验规则：两个日期均不可为空

- **传播服务类型**（必填）
  - 展示规则：标签组，每个标签含 emoji 图标 + 文字，单选
  - 数据规则：前端静态配置，固定排序（微博热搜 🔥、抖音热推 🎵、稿件热推 📝、央媒传播 📺、定制传播 🎯），从左到右、从上到下排列
  - 交互规则：选中后高亮为品牌蓝色背景，取消其他已选；切换类型时清空已选明细

- **传播服务明细**（必填，选择服务类型后展示）
  - 展示规则：单选卡片列表，每张卡片含标题 + 特性列表；单选
  - 数据规则：根据所选服务类型筛选，仅展示当前类型下的明细项；按配置顺序从上到下展示
  - 交互规则：选中后卡片边框变为品牌蓝色，显示勾选图标

- **提示信息**（选填后展示）
  - 展示规则：蓝色提示条，含信息图标 + 文字说明
  - 数据规则：固定文案「提交后，运营团队将在24小时内与您对接，为您匹配最佳传播方案并协调执行。」

- **服务类型与明细对照**：
  - 微博热搜
    - 同城热搜榜：冲击微博同城热搜 / 微博媒体号和KOL号同步参与发布 / 话题预计曝光量合计1000万
    - 全国热搜榜：冲击微博全国热搜 / 头部媒体账号联合发布 / 话题预计曝光量合计5000万+
  - 抖音热推
    - 同城热推：抖音同城热门推荐 / 本地KOL矩阵推广 / 预计播放量500万+
    - 全国热推：抖音全国热门推荐 / 头部达人联合推广 / 预计播放量2000万+
  - 稿件热推
    - 门户网站推送：新浪、网易、搜狐等门户首页 / 百度新闻源收录 / 预计阅读量100万+
    - 行业媒体推送：垂直行业头部媒体 / 精准触达目标受众 / 专业内容深度传播
  - 央媒传播
    - 央视报道：CCTV相关频道报道 / 权威媒体背书 / 全国范围传播
    - 新华社报道：新华社通稿发布 / 各大媒体转载 / 官方权威传播
  - 定制传播
    - 整合传播方案：多渠道整合传播 / 定制化传播策略 / 专属传播团队服务

#### Step 2：内容素材

**字段列表：**

- **文件上传**（选填）
  - 展示规则：虚线边框上传区域，含上传图标 + 文字说明
  - 数据规则：仅支持 docx、xlsx 格式，其他格式拒绝上传并提示；单文件不超过 5MB；最多 5 个文件；超长文件名截断显示
  - 交互规则：点击触发文件选择；上传成功后显示文件卡片（文件名 + 删除按钮）

- **任务描述**（必填）
  - 展示规则：多行文本域，placeholder「请详细描述您的传播需求...」
  - 数据规则：最大长度 500 字，超出截断，右下角实时显示「当前字数/500」
  - 校验规则：不可为空

#### Step 3：对接人信息

**字段列表：**

- **姓名**（必填）
  - 展示规则：单行文本输入框
  - 校验规则：不可为空

- **联系方式**（必填）
  - 展示规则：单行文本输入框，placeholder「请输入手机号码」
  - 数据规则：11 位纯数字，符合中国大陆手机号格式（1 开头）
  - 校验规则：不可为空，格式不合规时提示

- **职务**（选填）
  - 展示规则：单行文本输入框

- **公司**（选填）
  - 展示规则：单行文本输入框

**需求概览卡片：**
- 展示规则：展示前两步已填信息的摘要（任务名称、起止时间、服务类型、附件数量、任务描述），只读不可编辑
- 数据规则：未填写字段显示「-」

## 全局交互规则

### 步骤导航

- 顶部进度条展示 3 个步骤，当前步骤高亮
- 底部操作区：第 1 步仅显示「下一步」；第 2~3 步显示「上一步」+「下一步/提交需求」
- 按钮状态：必填字段未满足时按钮置灰不可点击

### 提交行为

- 提交前校验：所有必填字段非空 + 格式合规
- 点击「提交需求」后进入结果页，展示提交成功信息

### 结果页

- 展示规则：
  - 居中布局，浅蓝渐变背景（从品牌蓝 5% 到白色）
  - 蓝紫渐变圆形图标（含白色勾选），带弹出动画
  - 「提交成功」标题 + 副标题「提交后，运营团队将在24小时内与您对接，为您匹配最佳传播方案并协调执行。」
  - 中部白色卡片展示需求概览，含蓝色竖条标题「需求概览」
  - 需求概览卡片展示：任务名称、服务类型、对接人姓名
  - 底部「再填一个」按钮，品牌渐变背景，阴影效果
- 交互规则：
  - 点击「再填一个」清空所有表单字段，回到第 1 步重新填写
- 数据规则：
  - 需求概览卡片展示本次提交的任务名称、服务类型、对接人姓名
  - 未填写字段显示「-」
  - 任务名称超出宽度时截断显示

### 视觉规范

- 主色调：#1677FF（品牌蓝）
- 渐变：从 #1677FF 到 #4F46E5（蓝紫渐变）
- 圆角：卡片 16px，按钮 12px，输入框 12px
