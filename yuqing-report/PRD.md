# 舆情报送与管理系统 产品需求文档

> 版本：v1.2.0 | 日期：2026-04-22
> 上一版本：v1.1.4
> 原型文件：index.html

## 1. 产品概述

### 1.1 产品背景

省级宣传部门需要一套轻量化的舆情信息报送与管理工具，用于收集各地市上报的舆情事件、汇总查看、并编写舆情专报上报领导。本系统为简版，聚焦核心报送流程，去除数据分析、趋势预警等高级功能，降低使用门槛。

系统采用手机号密码登录方式，实现统一身份认证与权限管控。

### 1.2 产品目标

- 为各地市宣传部提供统一的舆情上报入口，规范报送流程
- 为舆情处提供舆情汇总查看、批量录用至报告的快捷操作
- 提供轻量的报告编写、预览与发布能力
- 实现组织内统一登录与权限管控

### 1.3 用户角色与权限

系统定义三种用户角色，权限严格区分：

- **地市人员**
  - 所属组织：各地市宣传部（如杭州市宣传部、温州市宣传部等）
  - 核心权限：上报本地舆情信息
  - 数据可见范围：仅可查看本单位上报的舆情记录
  - 可用功能模块：舆情汇总（限本单位数据）、舆情上报
  - 不可用功能：报告管理、录用至报告、系统管理

- **舆情处人员**
  - 所属组织：省委宣传部舆情处
  - 核心权限：舆情汇总管理与报告编写
  - 数据可见范围：全部地市上报舆情 + 外省镜鉴舆情
  - 可用功能模块：舆情汇总（全部数据）、报告管理（含录用、编辑、预览、发布）
  - 不可用功能：系统管理

- **系统管理员**
  - 所属组织：省委宣传部
  - 核心权限：系统运维与权限配置
  - 数据可见范围：全部
  - 可用功能模块：全部功能 + 系统管理（用户管理、角色管理、部门管理）

## 2. 登录认证

### 2.1 登录方式

系统提供手机号密码登录方式，登录流程如下：

- 用户打开系统后显示登录页
- 输入手机号、密码和验证码完成身份认证
- 系统校验手机号和密码匹配后，自动获取用户姓名、所属部门、角色等信息
- 根据用户在系统中的角色配置，自动进入对应权限的主界面
- 若不填写手机号和密码，直接点击登录可进入系统（原型演示模式）

### 2.2 登录页信息展示

- 左侧展示系统名称、功能简介、核心特性列表
- 右侧为登录操作区域，包含以下字段：
  - **手机号**：文本输入框，占位文字"请输入手机号"，限制 11 位
  - **密码**：密码输入框，占位文字"请输入密码"
  - **验证码**：文本输入框（限制 4 位）+ 右侧图形验证码画布
    - 展示规则：画布显示 4 位随机字符（大小写字母+数字），配干扰线和干扰点
    - 交互规则：点击画布刷新验证码内容
  - **登录按钮**：点击执行登录
- 底部显示体验提示信息

### 2.3 登录后初始化

- 自动填充用户信息至顶部导航栏（头像取姓名首字）
- 根据角色动态构建侧边栏导航菜单
- 默认进入"舆情汇总"页面

### 2.4 退出登录

- 顶部导航栏右侧提供退出按钮
- 点击后清除当前会话，返回登录页

## 3. 全局导航

### 3.1 侧边栏结构

侧边栏采用统一的导航结构，各菜单项的可见性根据用户角色通过配置控制：

- **顶部 Logo 区**
  - 系统名称"舆情报送与管理系统"，配蓝色圆角图标

- **功能菜单**
  - 舆情汇总：所有角色可见。地市人员仅可查看本单位上报数据，舆情处和管理员可查看全部数据并使用录用功能
  - 报告管理：仅舆情处和管理员可见。地市人员无此入口

- **系统管理**（仅管理员可见）
  - 部门管理
  - 用户管理
  - 角色管理

> 舆情上报功能通过舆情汇总页右上角按钮进入，不在侧边栏中单独展示。原型演示中默认展示全部菜单，实际系统根据用户角色动态控制菜单项的显示与隐藏。

### 3.2 顶部导航栏

- **左侧**
  - 当前页面标题，随页面切换动态更新

- **右侧**
  - 用户头像（姓名首字）
  - 用户姓名
  - 退出登录图标按钮

## 4. 舆情汇总

### 4.1 功能说明

查看系统中已上报的舆情信息列表。根据用户角色不同，数据范围和操作权限有显著差异：

- 地市人员：仅展示本单位上报的舆情记录，无录用功能
- 舆情处/管理员：展示全部上报舆情，支持多选后批量录用至报告

### 4.2 统计概览

页面标题下方展示 4 张统计卡片，实时反映当前筛选结果。

- **取数规则**：统计基于当前筛选条件过滤后的舆情列表
- **更新规则**：筛选条件变化时实时重新计算

各卡片字段：

1. **上报总数**
   - 展示规则：蓝色图标，显示数字 + "上报总数"标签
   - 数据规则：当前筛选结果中的舆情总条数

2. **已录用**
   - 展示规则：橙色图标，显示数字 + "已录用"标签
   - 数据规则：当前筛选结果中已被录用至任意报告的舆情条数（即录用状态为"已录用"）

3. **本省**
   - 展示规则：蓝色图标，显示数字 + "本省"标签
   - 数据规则：当前筛选结果中来源类型为"本省"的舆情条数

4. **外省镜鉴**
   - 展示规则：绿色图标，显示数字 + "外省镜鉴"标签
   - 数据规则：当前筛选结果中来源类型为"外省镜鉴"的舆情条数

### 4.3 搜索与筛选

#### 4.3.1 搜索框

- 展示规则：文本输入框，占位文字"搜索标题、地域、单位、内容..."
- 交互规则：输入后实时过滤列表，清空搜索框恢复全部数据
- 数据筛选规则：按舆情标题、地域、上报单位、内容摘要四个字段进行模糊匹配（包含即命中），取并集

#### 4.3.2 时间范围筛选

- 展示规则：两个日期选择器，组成"起始日期 至 结束日期"
- 交互规则：选择后立即过滤，可单独设置起始或结束，另一端不限
- 数据筛选规则：按舆情的上报时间字段进行范围匹配，仅显示上报时间在起止日期之间的舆情

#### 4.3.3 来源类型筛选

- 展示规则：下拉选择框
- 交互规则：选择后立即过滤列表
- 数据筛选规则：
  - 默认值："全部来源"（不限）
  - 选项："全部来源"、"本省"、"外省镜鉴"
  - 选中后按舆情的来源类型字段精确匹配

#### 4.3.4 单位类型筛选

- 展示规则：下拉选择框
- 交互规则：选择后立即过滤列表
- 数据筛选规则：
  - 默认值："全部单位"（不限）
  - 选项及匹配方式：
    - "省级"：上报单位包含"省委"或"省网信"
    - "地市"：上报单位包含"市宣传部"

#### 4.3.5 录用状态筛选

- 展示规则：下拉选择框
- 交互规则：选择后立即过滤列表
- 数据筛选规则：
  - 默认值："录用状态"（不限）
  - 选项及匹配方式：
    - "已录用"：舆情已被至少一个报告收录
    - "未录用"：舆情未被任何报告收录

#### 4.3.6 地域筛选

- 展示规则：二级级联控件（省→市），第一级省份下拉，第二级城市下拉（选择省份后显示）
- 交互规则：选择后立即过滤，支持仅选省份或选省份+城市
- 数据筛选规则：
  - 第一级含"全部省份"选项，选具体省份后按舆情地域的省份字段精确匹配
  - 第二级含"全部城市"选项，选具体城市后按舆情地域的城市字段精确匹配
  - 两级均选时取交集

#### 4.3.7 舆情分类筛选

- 展示规则：下拉选择框
- 交互规则：选择后立即过滤列表
- 数据筛选规则：
  - 默认值："全部分类"（不限）
  - 选项："刑事案件"、"治安事件"、"交通事故"、"执法不当及违纪违法"、"非正常死亡或失踪"、"其他"
  - 选中后按舆情的分类字段精确匹配，未打标分类的舆情不显示

#### 4.3.8 筛选组合规则

- 搜索关键词、时间范围、来源类型、单位类型、录用状态、地域、舆情分类七个筛选条件可同时生效，取交集
- 所有筛选条件变化后实时更新列表和统计数据

### 4.4 操作栏

- **全选按钮**（仅舆情处/管理员可见）
  - 展示规则：按钮文字"全选"/"取消全选"
  - 交互规则：点击后勾选/取消当前筛选结果中的所有舆情，仅对当前可见列表生效
  - 数据规则：不包含被筛选条件隐藏的条目

- **信息计数**
  - 展示规则：文字"共 N 条上报信息"
  - 数据规则：N = 当前筛选结果的总条数，随筛选条件变化实时更新

### 4.5 舆情列表

- **取数规则**：从舆情数据集中读取。地市人员仅读取上报单位与本单位匹配的记录，舆情处和管理员读取全部记录
- **数据筛选规则**：在角色数据范围基础上，叠加 4.3 中所有筛选条件，取交集
- **数据排序规则**：默认按上报时间倒序排列（最新在上）

#### 4.5.1 列表项展示字段

每条舆情以卡片形式展示，从左到右包含：

1. **勾选框**（仅舆情处/管理员可见）
   - 展示规则：圆形复选框
   - 交互规则：点击切换选中/取消，选中后卡片左侧出现蓝色边框，背景变为浅蓝色

2. **舆情元信息行**
   - **舆情编号**：展示规则：蓝色文字。数据规则：系统自动生成，格式 YQ-YYYYMMDD-NNN（NNN 为当日序号，从 001 递增）
   - **来源类型标签**：展示规则：本省为蓝色浅底标签，外省镜鉴为橙色浅底标签。数据规则：取舆情的来源类型字段
   - **地域**：展示规则：地市名称，配地理位置图标。数据规则：取舆情的地域字段（省+市拼接）
   - **分类标签**（已打标时显示）：展示规则：绿色浅底标签。数据规则：取舆情的分类字段，未打标不显示
   - **录用状态标签**：展示规则：已录用显示绿色浅底标签，未录用显示灰色浅底标签。数据规则：判断该舆情是否被任意报告收录

3. **舆情标题**
   - 展示规则：单行显示，超出部分省略号
   - 交互规则：点击进入舆情详情页

4. **底部信息行**
   - **上报单位**：展示规则：配建筑图标。数据规则：取舆情的上报单位字段
   - **上报时间**：展示规则：格式 YYYY-MM-DD HH:mm，配时钟图标。数据规则：取舆情的上报时间字段

#### 4.5.2 空状态

- 无数据时展示"未找到匹配的舆情信息"提示，配大号图标辅助说明

### 4.6 舆情分类打标

拥有权限的用户（舆情处和管理员）可对舆情进行分类打标。

- 展示规则：每条舆情卡片右侧显示打标按钮（标签图标）
- 交互规则：
  1. 点击后弹出分类选择弹窗，以单选列表形式展示分类选项
  2. 已有分类的舆情默认选中当前分类
  3. 选择后点击"确认"完成打标
  4. 支持重复打标覆盖原有分类
- 数据规则：
  - 分类选项固定为："刑事案件"、"治安事件"、"交通事故"、"执法不当及违纪违法"、"非正常死亡或失踪"、"其他"
  - 确认后将选中的分类写入该舆情的分类字段
  - 打标后舆情卡片元信息行即时显示绿色分类标签

### 4.7 录用至报告交互流程

> 仅舆情处和管理员可操作

- 交互规则：
  1. 勾选一条或多条舆情（通过列表勾选框或全选按钮）
  2. "录用至报告"按钮从灰色变为蓝色可点击态
  3. 点击后弹出报告选择弹窗
  4. 用户选择目标报告后点击"确认录用"
  5. 系统将选中舆情添加到目标报告的舆情列表中（已存在的不会重复添加）
  6. 弹窗关闭，显示成功提示
  7. 自动跳转至该报告的编辑页面
- 数据规则（弹窗报告列表）：
  - 取数规则：从报告数据集中读取所有状态为"草稿"的报告
  - 数据排序规则：按创建时间倒序排列
  - 每项展示：报告编号 + 报告标题

## 5. 舆情上报

### 5.1 功能说明

地市人员通过此页面上报本地舆情信息。支持一次填写多条舆情，批量提交。

> 舆情处和管理员也可使用此功能（通过汇总页右上角按钮进入）。

### 5.2 多条上报机制

- 展示规则：页面默认展示 1 条舆情填写表单，每条为独立卡片，标题显示"第 N 条舆情"
- 交互规则：
  - 点击"添加一条舆情"按钮在下方新增一条空白表单
  - 第 1 条不显示移除按钮，第 2 条及以后每条右上角显示移除按钮
  - 点击移除按钮后该卡片以滑出动画消失
  - 页面右上角实时显示当前表单总数："共 N 条"
  - 底部提交按钮同步显示条数："提交上报（N 条）"

### 5.3 单条表单字段

每条舆情表单包含以下字段：

1. **上报时间**
   - 展示规则：日期时间选择器，精确到分钟
   - 交互规则：必填，未填时边框变红
   - 数据规则：写入舆情记录的上报时间字段

2. **舆情地域**
   - 展示规则：二级级联控件（省→市），第一级省份下拉，第二级城市下拉（选择省份后启用）
   - 交互规则：必填，选择后自动触发"报送单位"字段的填充
   - 数据规则：写入舆情记录的地域字段（省+市）

3. **报送单位**
   - 展示规则：下拉选择框，默认选中当前用户所属单位
   - 交互规则：非必填，用户可从下拉列表中手动切换报送单位
   - 取数规则：下拉选项取数来源为部门树中所有末级部门（地市宣传部）

4. **标题**
   - 展示规则：文本输入框，占位文字"请输入舆情标题"
   - 交互规则：必填，未填时边框变红
   - 数据规则：写入舆情记录的标题字段

5. **内容**
   - 展示规则：多行文本输入框，占位文字"请详细描述舆情事件经过、传播情况、影响范围等..."
   - 交互规则：必填，未填时边框变红
   - 数据规则：写入舆情记录的内容摘要字段

### 5.4 操作按钮

位于所有表单卡片下方：

1. **添加一条舆情**
   - 展示规则：轮廓样式按钮，带加号图标
   - 交互规则：点击后在最下方新增一条空白表单

2. **重置**
   - 展示规则：轮廓样式按钮，带刷新图标
   - 交互规则：点击后清空所有已填写的表单卡片，恢复为仅 1 条空白表单的初始状态

3. **提交上报（N 条）**
   - 展示规则：主按钮样式，带发送图标，N 随当前表单数量动态变化
   - 交互规则：
     - 点击后逐一校验所有表单中的必填字段（上报时间、地域、标题、内容）
     - 未填写的必填字段边框变红，弹出错误提示指明需完善第几条表单
     - 校验通过后弹出成功提示，显示实际上报条数，延迟后自动跳转回舆情汇总页
   - 数据规则：
     - 舆情编号生成规则：格式 YQ-YYYYMMDD-NNN，YYYYMMDD 取上报日期，NNN 为当日序号（从 001 递增）
     - 来源类型：根据上报用户所属单位自动判定（地市人员上报的为"本省"）
     - 录用状态：默认"未录用"
     - 分类字段：默认为空（待后续打标）
     - 写入舆情数据集

## 6. 舆情详情

### 6.1 功能说明

点击舆情列表中的标题，进入该条舆情的完整信息展示页。

- 取数规则：根据当前点击的舆情编号，从舆情数据集中读取该条记录的全部字段

### 6.2 页面顶部

- "返回列表"按钮，点击回到舆情汇总页

### 6.3 信息展示

**元信息行**
- 舆情编号：蓝色文字，取舆情编号字段
- 来源类型标签：本省/外省镜鉴，取来源类型字段
- 地域：配图标，取地域字段

**舆情标题**
- 大号加粗显示，取标题字段

**舆情内容区**
- "舆情内容"小标题
- 完整摘要内容，灰色背景区域展示，取内容摘要字段

**基本信息区**
- 上报单位：取上报单位字段
- 上报时间：取上报时间字段，格式 YYYY-MM-DD HH:mm

### 6.4 操作按钮

- **"录用至报告"按钮**（仅舆情处/管理员可见）
  - 展示规则：蓝色主按钮
  - 交互规则：点击后触发录用流程（自动勾选当前这一条，弹出报告选择弹窗），弹窗逻辑同 4.7

- **"返回列表"按钮**
  - 展示规则：轮廓样式，带左箭头图标
  - 交互规则：点击回到舆情汇总页

## 7. 报告管理

### 7.1 功能说明

舆情处和管理员在此编写、预览和管理舆情报告。报告由多条舆情信息按【内容提要】+ 一、本省舆情 + 二、外省舆情镜鉴的结构组合而成。

> 地市人员无此模块入口。

### 7.2 报告列表

- 取数规则：从报告数据集中读取所有报告记录
- 数据筛选规则：舆情处和管理员可查看全部报告
- 数据排序规则：按创建时间倒序排列（最新在上）

#### 7.2.1 页面标题区

- 标题"报告管理"
- 右侧"新建报告"按钮（蓝色主按钮）

#### 7.2.2 报告卡片展示字段

每条报告以卡片形式展示：

1. **报告编号**
   - 展示规则：蓝色文字
   - 数据规则：系统自动生成，格式 BG-YYYYMMDD-NNN（NNN 为当日序号，从 001 递增）

2. **报告状态标签**
   - 展示规则：草稿为灰色标签，已发布为绿色标签
   - 数据规则：取报告的状态字段

3. **报告标题**
   - 展示规则：单行显示
   - 数据规则：取报告的标题字段

4. **底部信息**
   - 创建时间：展示规则：日期时间格式。数据规则：取报告的创建时间字段
   - 包含舆情条数：展示规则：显示"N 条舆情"。数据规则：该报告收录的舆情总数

#### 7.2.3 操作按钮显示规则

根据报告状态显示不同的按钮组合：

- **草稿状态**
  - "编辑"按钮（蓝色小按钮）：进入报告编辑页
  - "预览"按钮（轮廓小按钮）：进入报告预览页
  - 删除按钮（红色图标按钮）：删除报告（需二次确认）

- **已发布状态**
  - "预览"按钮（轮廓小按钮）：进入报告预览页

#### 7.2.4 新建报告

- 交互规则：点击"新建报告"按钮，创建后立即进入报告编辑页
- 数据规则：
  - 报告编号：系统自动生成，格式 BG-YYYYMMDD-NNN
  - 报告标题：默认为空，由用户在编辑页手动输入
  - 状态：默认"草稿"
  - 舆情列表：默认为空
  - 创建时间：取当前时间
  - 写入报告数据集

### 7.3 报告预览

#### 7.3.1 入口

- 点击报告管理列表中的报告卡片（任意状态）
- 点击报告卡片上的"预览"按钮
- 从报告编辑页点击"预览"按钮

#### 7.3.2 页面布局

- 顶部操作栏：
  - 左侧"返回"按钮
  - 右侧按钮组根据报告状态不同：
    - 草稿：显示"编辑"按钮 + "导出 Word"按钮
    - 已发布：显示"导出 Word"按钮

#### 7.3.3 预览内容格式（公文格式）

- 取数规则：根据当前报告 ID 从报告数据集中读取报告信息及其收录的舆情列表
- 数据排序规则：先展示"本省"来源的舆情，再展示"外省镜鉴"来源的舆情；各来源内部按舆情编号正序排列

预览以正式公文样式展示，结构如下：

1. **标题区**
   - 居中显示报告标题（大号加粗，字间距加大）
   - 下方显示报告日期
   - 底部双横线分隔

2. **【内容提要】**（目录样式，小号字体）
   - 一、省内舆情：列出所有"本省"类型舆情的序号 + 标题
   - 二、外省舆情镜鉴：列出所有"外省镜鉴"类型舆情的序号 + 标题
   - 数据规则：序号按各分组内顺序从 1 递增；若无外省舆情则不显示外省子板块

3. **一、本省舆情**
   - 每条"本省"舆情独立一段：
     - 序号 + 标题（加粗）
     - 完整摘要内容
     - 建议意见（如有，橙色图标 + 灰色文字）
     - 来源和时间（灰色小字）

4. **二、外省舆情镜鉴**
   - 每条"外省镜鉴"舆情独立一段，格式与本省舆情一致
   - 若无外省舆情则不显示此板块

### 7.4 报告编辑

#### 7.4.1 入口

- 草稿报告的"编辑"按钮
- 新建报告后自动进入
- 从舆情汇总页录用舆情后自动跳转

#### 7.4.2 页面布局

页面分为左右两栏：

**左栏 — 编辑区**

顶部操作按钮栏（工具条样式，左对齐操作按钮 + 右对齐发布按钮）：
- 左侧："选取舆情"按钮（轮廓样式）：弹出舆情选择弹窗
- 左侧："保存草稿"按钮（轮廓样式）：保存当前编辑内容
- 右侧："预览"按钮（蓝色主按钮）：跳转预览页
- 右侧："发布"按钮（绿色主按钮）：发布报告

顶部标题栏：
- 左侧："返回"按钮 + 报告编辑状态标题
- 右侧：统计标签（"本省 N" + "外省 N"）

基本信息区：
- **报告标题**（必填）：文本输入框，默认取自动生成的日期标题
- **报告日期**：日期选择器

统计标签：
- "本省: N"（蓝色浅底标签）：数据规则 = 当前报告中来源为"本省"的舆情条数
- "外省: N"（橙色浅底标签）：数据规则 = 当前报告中来源为"外省镜鉴"的舆情条数

舆情列表分为两个区块（每个区块标题带蓝色竖线装饰）：

- **本省舆情（N）**
  - 每条舆情以可编辑卡片形式展示
  - 卡片头部：序号 + 编号 + 来源类型标签 + 删除按钮（红色图标）
  - 卡片内容：
    - 标题输入框：可直接修改标题文字
    - "正文内容"文本区：可直接修改正文，默认取舆情摘要
    - "建议意见"文本区：可填写建议

- **外省舆情镜鉴（N）**
  - 结构与本省舆情相同
  - 无外省舆情时不显示此区块

**右栏 — 实时预览面板**

- **【内容提要】预览**
  - 一、省内舆情：所有本省舆情的序号 + 标题
  - 二、外省舆情镜鉴：所有外省舆情的序号 + 标题
  - 若无外省舆情则不显示外省子板块

> 右侧预览随左侧编辑内容实时更新。

#### 7.4.3 编辑交互

- **修改标题**：在卡片中的标题输入框直接编辑，右侧预览面板同步更新
- **修改正文**：在"正文内容"文本区直接编辑
- **修改建议意见**：在"建议意见"文本区直接编辑
- **删除舆情**：点击卡片右上角的删除按钮，该条目从报告中移除，列表和预览同步更新
- **添加舆情**：
  1. 点击"选取舆情"按钮
  2. 弹窗展示当前未收录到本报告的所有舆情列表
  3. 每条展示来源类型标签、地域、标题
  4. 支持多选（复选框）
  5. 点击"确认"后，选中舆情添加到报告的舆情列表中
  6. 已存在的舆情不会重复添加
  - 数据规则（弹窗舆情列表）：
    - 取数规则：从舆情数据集中读取所有未被当前报告收录的舆情
    - 数据排序规则：按上报时间倒序排列

#### 7.4.4 保存与发布

- **保存草稿**
  - 交互规则：将当前编辑内容保存，状态保持"草稿"，弹出成功提示
  - 数据规则：将报告标题、报告日期、舆情列表（含修改后的标题/正文/建议意见）写入报告数据集

- **发布**
  - 交互规则：校验报告标题是否为空，为空则提示错误；校验通过后弹出成功提示，自动跳转回报告管理列表
  - 数据规则：报告状态字段更新为"已发布"

- **预览**
  - 交互规则：先自动保存草稿，跳转至公文格式预览页

## 8. 系统管理

> 仅系统管理员可访问此模块。

### 8.1 用户管理

#### 8.1.1 功能说明

管理系统中的用户账号。支持新增、编辑用户，角色分配与状态管理。

#### 8.1.2 页面标题区

- 标题"用户管理"
- 右侧"新增用户"按钮

#### 8.1.3 用户列表

- 取数规则：从用户数据集中读取所有用户记录
- 数据筛选规则：默认展示全部用户，无额外筛选
- 数据排序规则：按用户 ID 正序排列

以表格形式展示所有用户，列字段如下：

1. **姓名**
   - 展示规则：普通文字
   - 数据规则：取用户的姓名字段

2. **手机号**
   - 展示规则：普通文字
   - 数据规则：取用户的手机号字段

3. **所属部门**
   - 展示规则：普通文字，多个部门以分号（；）分隔展示
   - 数据规则：取用户关联的部门名称列表

4. **角色**
   - 展示规则：多个角色名称以分号（；）分隔展示
   - 数据规则：取用户被分配的角色名称列表

5. **状态**
   - 展示规则：滑动开关，启用显示绿色圆点+绿色文字"正常"，停用显示灰色圆点+灰色文字"停用"
   - 交互规则：点击切换启用/停用，切换后立即生效，显示成功提示
   - 数据规则：取用户的状态字段（"正常"或"停用"）

6. **操作**
   - 展示规则："编辑"按钮（蓝色小按钮）
   - 交互规则：点击弹出编辑弹窗，修改用户姓名、手机号、所属部门和角色分配

#### 8.1.4 新增用户

- 交互规则：
  1. 点击"新增用户"按钮弹出弹窗
  2. 填写表单字段（见下方）
  3. 校验通过后确认添加
- 弹窗表单字段：
  - **姓名**：文本输入框，必填
  - **手机号**：文本输入框，必填，校验 11 位格式，不可与已有用户重复
  - **密码**：密码输入框，必填，至少 8 个字符，必须包含大写字母、小写字母、数字、符号（!@#$%^&*?）
  - **确认密码**：密码输入框，必填，需与密码一致
  - **所属部门**：复选框多选，至少选择一个
    - 数据规则：取数来源为部门树中所有末级部门，选项以"省 / 市 / 县"路径形式展示
  - **角色**：复选框多选，至少选择一个
- 数据规则：
  - 确认后新增到用户数据集
  - 用户 ID：自动生成
  - 状态：默认"正常"
  - 密码：取用户填写的密码值
  - 同时在每个选中部门的人员列表中添加该用户

#### 8.1.5 编辑用户

- 交互规则：
  1. 点击用户列表中的"编辑"按钮弹出弹窗
  2. 修改表单字段后保存
- 弹窗表单字段：
  - **姓名**：文本输入框，预填当前姓名
  - **手机号**：文本输入框，预填当前手机号
  - **重置密码**：密码输入框，非必填
    - 展示规则：占位文字"留空则不修改密码"，下方灰色提示"留空表示不修改密码，填写需至少8位且包含大写、小写、数字、符号"
    - 交互规则：留空则不修改密码，填写则校验至少 8 位且包含大写字母、小写字母、数字、符号（!@#$%^&*?）
  - **所属部门**：复选框多选，预选当前已关联的部门
  - **角色**：复选框多选，预选当前角色

### 8.2 角色管理

#### 8.2.1 功能说明

查看和配置系统角色及其功能权限。系统预置三种角色，每种角色对应不同的功能权限范围。支持新增自定义角色、删除角色、编辑角色信息、分配功能权限。

#### 8.2.2 页面布局

页面采用左右分栏布局：

**左栏 — 角色列表**

- 取数规则：从角色数据集中读取所有角色
- 数据筛选规则：搜索框支持按角色名称和描述模糊匹配
- 数据排序规则：按角色 ID 正序排列

- 顶部区域：
  - 标题"角色列表"
  - "新增"按钮（蓝色小按钮）
  - 搜索框：支持按角色名称和描述模糊搜索
- 角色列表：
  - 每项显示角色名称
  - 鼠标悬停时右侧显示删除按钮（红色图标）
  - 当前选中项蓝色背景高亮
  - 点击切换右侧内容

**右栏 — 角色详情（自适应宽度）**

通过顶部标签页切换三个视图：

1. **角色编辑**标签页
   - 角色名称：文本输入框，修改后左侧列表实时同步
   - 角色描述：文本输入框

2. **功能权限分配**标签页
   - 展示规则：以树形结构展示系统功能模块及子权限，每个节点前有复选框
   - 交互规则：
     - 父节点勾选：子节点全部勾选
     - 子节点全部勾选：父节点自动勾选
     - 子节点部分勾选：父节点显示半选状态（横线图标）
     - 底部"保存"按钮保存权限配置
   - 权限树结构：
     - 舆情汇总：舆情上报、录用至报告、舆情分类打标
     - 报告管理：新建报告、编辑报告、删除报告、预览报告、发布报告
     - 系统管理：用户管理、角色管理、部门管理

3. **用户管理**标签页
   - 取数规则：从用户数据集中读取属于当前角色的用户列表
   - 数据筛选规则：搜索框支持按姓名、所属部门模糊匹配
   - 数据排序规则：按用户 ID 正序排列
   - 顶部操作区：
     - 左侧："添加用户"按钮（蓝色小按钮）：弹出弹窗，多选未属于当前角色的用户，确认后加入当前角色
     - 左侧："移除用户"按钮（轮廓小按钮）：勾选用户后出现，点击后弹出确认弹窗，批量移除选中用户
     - 右侧：搜索框
   - 用户列表以表格形式展示，列字段：
     - 勾选框：支持单选和全选，选中后"移除用户"按钮出现
     - 姓名
     - 所属部门
     - 状态标签：已开通（绿色）、已停用（橙色）
     - 操作："移除"链接，点击弹出确认弹窗后移除单个用户
   - 底部显示用户总数
   - 无用户时展示空状态提示

#### 8.2.3 新增角色

- 交互规则：点击左栏"新增"按钮弹出弹窗，填写角色名称（必填）和角色描述，确认后添加
- 数据规则：新增到角色数据集，权限默认为空

#### 8.2.4 删除角色

- 交互规则：鼠标悬停角色项时显示删除按钮，点击后弹出确认弹窗
- 数据规则：从角色数据集中删除该角色。系统至少保留一个角色，不可全部删除

#### 8.2.5 预置角色权限配置

1. **地市人员**
   - 描述：各地市宣传部工作人员
   - 权限：舆情上报

2. **舆情处人员**
   - 描述：省委宣传部舆情处工作人员
   - 权限：舆情上报、录用至报告、舆情分类打标、新建报告、编辑报告、预览报告、发布报告

3. **系统管理员**
   - 描述：系统运维管理员
   - 权限：全部功能权限

### 8.3 部门管理

#### 8.3.1 功能说明

管理系统的组织架构。管理员可手动创建、编辑、删除部门层级结构，部门数据作为用户管理中"所属部门"复选框选项的数据源，也作为角色管理中"添加用户"弹窗的人员数据源。

- 取数规则：从部门数据集中读取完整的树形结构数据

#### 8.3.2 页面标题区

- 标题"部门管理"
- 右侧"新增部门"按钮（蓝色小按钮）：点击后弹窗填写部门名称，确认后添加到组织树顶层

#### 8.3.3 统计概览

无统计概览卡片。

#### 8.3.4 组织树展示

以可展开/折叠的树形结构展示组织架构，最多支持两级层级，支持动态增删改：

- 展示规则：
  - 第一级：如"浙江省""杭州市"，左侧文件夹图标，展开后显示下属部门
  - 第二级：如"杭州市宣传部""萧山区宣传部"，为末级节点，直接展示该部门下的人员
  - 不支持在第二级下继续添加子部门

每个层级节点展示：

- 展开/折叠箭头（仅第一级节点）
- 节点名称
- 操作按钮组：
  - "添加部门"图标按钮：仅第一级节点显示，点击后弹窗填写子部门名称，确认后在该节点下新增子部门
  - "编辑"图标按钮：点击后弹窗修改部门名称
  - "删除"图标按钮：点击后弹出二次确认弹窗，确认后删除该部门及其子部门

末级节点展示：

- 部门名称（配建筑图标）
- 操作按钮组（编辑、删除）
- 人员列表：每人以头像胶囊形式展示（头像首字 + 姓名），停用人员标注"(停用)"

#### 8.3.5 部门操作规则

- **新增部门**：
  1. 点击页面标题区"新增部门"按钮
  2. 弹窗填写部门名称（必填）
  3. 确认后添加到部门数据集顶层，自动生成唯一 ID

- **添加子部门**：
  1. 点击任意节点上的"添加部门"图标按钮
  2. 弹窗填写子部门名称（必填）
  3. 确认后添加到该节点的子节点列表中，自动生成唯一 ID
  4. 支持在任意层级节点下添加子部门，不限层级深度

- **编辑部门**：
  1. 点击节点上的"编辑"图标按钮
  2. 弹窗显示当前部门名称，可修改
  3. 确认后更新部门数据集中的部门名称

- **删除部门**：
  1. 点击节点上的"删除"图标按钮
  2. 弹出二次确认弹窗，提示该部门下的子部门和人员将一并移除
  3. 确认后从部门数据集中删除该部门及其所有子部门

#### 8.3.6 数据使用场景

- **新增用户**弹窗中的"所属部门"复选框选项：取数来源为部门数据集中所有末级部门，选项以路径形式展示
- **编辑用户**弹窗中的"所属部门"复选框选项：取数来源同上
- **角色管理-添加用户**弹窗中的可选人员列表：取数来源为部门数据集中所有人员，按省市分组展示，每条显示姓名头像 + 姓名 + 所属部门路径

## 10. 全局交互规则

### 10.1 消息提示

- **操作成功**：顶部绿色轻提示，显示成功图标和操作结果文字，2.5 秒后自动消失
- **操作失败**：顶部红色轻提示，显示错误图标和原因文字，2.5 秒后自动消失
- **信息提示**：顶部蓝色轻提示，用于功能开发中等提示场景

### 10.2 确认弹窗

- 用于不可逆操作的二次确认（如删除报告、删除部门）
- 包含标题、说明文字、"取消"和"确认"两个按钮
- 点击遮罩层可关闭弹窗
- 操作按钮右对齐，确认按钮在右侧

### 10.3 页面切换

- 侧边栏导航点击切换页面，当前激活项以蓝色背景高亮
- 顶部导航栏标题随页面切换同步更新
- 页面内容区滚动位置自动回到顶部
