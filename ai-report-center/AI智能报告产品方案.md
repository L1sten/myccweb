# 舆情系统 AI 智能报告 — 产品方案

> 版本：v1.0 | 日期：2026-04-07

---

## 一、产品定位

### 1.1 一句话定义

基于监测方案采集的全量舆情数据，自动生成行业日常呈报报告的 AI 能力。

### 1.2 核心价值

| 角色 | 痛点 | 解决方式 |
|------|------|---------|
| 舆情分析师 | 每期报告耗时4-8小时，大量统计和编排工作 | AI 自动完成数据统计、事件梳理、内容撰写 |
| 部门领导 | 需要快速了解本期舆情全貌 | 自动生成结构化报告，30分钟内可出初稿 |
| 系统管理员 | 不同行业客户报告格式不同 | 行业预设模板，一次配置长期复用 |

### 1.3 设计原则

- **复用现有能力**：基于现有报告中心（report_template / report_letters）扩展，不重建
- **固定骨架 + 行业预设**：五章结构固定，通过行业模板提供领域分类
- **系统预处理 + 大模型生成**：系统侧做数据加工和统计，大模型侧做内容生成
- **人工可干预**：AI 生成初稿，人工可在线编辑修改后导出

## 二、报告结构设计

### 2.1 固定五章骨架

所有行业报告统一遵循以下结构，不可配置、不可调整顺序：

```
┌─────────────────────────────────────┐
│ 封面                                 │
│   期号 / 发布单位 / 日期 / 监测时段   │
├─────────────────────────────────────┤
│ 一、舆情概述                         │
│   信息总量统计 + 来源分布            │
│   排名变化 + 各领域动态 + 敏感总判断  │
├─────────────────────────────────────┤
│ 二、热点事件舆情分析                  │
│   按热度排序的事件逐条分析            │
├─────────────────────────────────────┤
│ 三、敏感信息舆情分析                  │
│   负面/敏感事件逐条分析              │
├─────────────────────────────────────┤
│ 四、本期重点关注                     │
│   AI 自动推荐的深度专题分析           │
├─────────────────────────────────────┤
│ 五、行业热点资讯                     │
│   行业相关新闻逐条摘要               │
└─────────────────────────────────────┘
```

### 2.2 敏感度判断（固定规则）

直接复用现有系统的情感标签体系：

- **正面** / **中性** → 归入"热点事件"或"行业资讯"
- **负面** → 归入"敏感信息舆情分析"

无需额外配置，基于 ES 字段 `n_sentiment_class` 直接判断。

### 2.3 重点关注选题（纯数据驱动）

不预设行业规则，由以下数据信号自动决定：
- 热搜关联（`hot_search` / `hot_search_event` 表，在榜时长、最高排名）
- 事件热度突变（声量环比增长 > 200%）
- 跨事件关联（多个事件指向同一深层议题）

系统侧提供候选主题 + 素材包，大模型侧完成深度写作。

### 3.2 领域分类的作用

领域分类在报告中出现在两个位置：
1. **舆情概述**：按领域分组概述各领域动态（如"运输方面：... 铁路方面：..."）
2. **行业热点资讯**：按领域分组展示行业新闻

### 3.3 存储方式

扩展现有 `report_template` 表，增加字段：

```sql
ALTER TABLE report_template ADD COLUMN industry_type VARCHAR(50) COMMENT '行业类型';
ALTER TABLE report_template ADD COLUMN industry_domains JSON COMMENT '行业领域分类';
ALTER TABLE report_template ADD COLUMN report_config JSON COMMENT '报告配置';
```

`industry_domains` 示例：
```json
["铁路", "公路", "水运", "民航", "城市交通"]
```

`report_config` 示例：
```json
{
  "report_name": "江苏交通运输网络舆情监测",
  "publisher": "江苏省交通运输厅",
  "period_type": "weekly",
  "start_issue": 1,
  "max_hot_events": 5,
  "max_sensitive_events": 5,
  "max_industry_news": 10,
  "focus_topic_source": "auto",
  "export_formats": ["docx", "pdf"]
}
```

## 五、数据接口设计（对接大模型团队）

### 5.1 现有可复用接口

| 现有接口 | 用途 | 数据来源 |
|---------|------|---------|
| 信息列表接口 | 拉取监测方案在时段内的全部信息 | ES (`monitor_column` 关联) |
| 信息趋势接口 | 获取声量趋势数据 | `situation_statistic` 表 |
| 舆论场统计 | 来源分布（新闻/微信/微博/...） | `situation_statistic` 表 |
| 敏感占比 | 正面/中性/负面占比 | ES `n_sentiment_class` |
| 行业分布 | 领域分类分布 | ES `info_class` |
| 热词统计 | 高频关键词 | ES `n_content_keywords` |
| 地域分布 | 省市级地域分布 | ES `n_content_areacode` |
| 热搜列表 | 涉浙/事件热搜数据 | `hot_search` / `hot_search_event` |
| 热搜趋势 | 热搜排名变化 | `hot_search_trend` / `hot_search_event_trend` |
| 报告生成 | 保存生成的报告 | `report_letters` (`addAutoReport`) |

### 5.3 分工边界

```
┌──────────────────────────────────────────────────────────┐
│                     系统侧（政企团队）                         │
│                                                          │
│  1. 数据拉取：从 ES + MySQL 拉取监测方案的时段数据         │
│  2. 统计聚合：总量、来源分布、趋势、排名                    │
│  3. 事件聚类：基于标题相似度聚类（6万条→40-50个事件）       │
│  4. 热度评分：article_count × source_weight × recency     │
│  5. 领域分类：基于 ES info_class 字段分组                  │
│  6. 敏感筛选：n_sentiment_class = "负面"                  │
│  7. 重点选题：热搜数据 + 趋势突变 + 事件关联 → 候选主题    │
│  8. 素材打包：将以上结构化数据组装为 JSON                   │
│  9. 报告存储：接收大模型返回的 HTML，存入 report_letters    │
└──────────────────────────────────────────────────────────┘
                         │
                    预处理数据 JSON
                         │
                         ▼
┌──────────────────────────────────────────────────────────┐
│                   大模型侧（大模型团队）                    │
│                                                          │
│  1. 接收预处理数据 JSON                                   │
│  2. 舆情概述：生成概述段落（统计描述+热点/敏感摘要）         │
│  3. 热点分析：为每个热点事件生成分析段落                     │
│  4. 敏感分析：为每个敏感事件生成分析段落                     │
│  5. 重点关注：基于候选主题素材包，生成深度专题文章            │
│  6. 行业资讯：为每条新闻生成摘要                            │
│  7. 自检校验：核对数据一致性                                │
│  8. 组装 HTML：按报告模板结构组装完整 HTML                   │
│  9. 返回 HTML                                            │
└──────────────────────────────────────────────────────────┘
```

## 六、产品流程

### 6.1 配置流程（一次性）

```
管理员进入 AI报告中心
    │
    ├─ 1. 选择行业类型（从10个预设中选择）
    │     → 自动填入领域分类
    │
    ├─ 2. 填写基本信息
    │     · 报告名称
    │     · 发布单位
    │     · 生成周期（日报/周报/月报）
    │     · 起始期号
    │
    ├─ 3. 关联监测方案
    │     · 从已有监测方案列表中选择
    │
    ├─ 4. 选择导出格式
    │     · Word / PDF
    │
    └─ 5. 保存配置 → 后续自动沿用
```

### 6.2 生成流程（每期）

```
触发方式：
  · 定时触发（按配置的周期自动执行）
  · 手动触发（点击"立即生成"）

生成流程：
  ┌──────────┐
  │ 触发生成  │
  └─────┬────┘
        ▼
  ┌──────────┐     ┌──────────────────────────────┐
  │ 数据准备  │ ──→ │ 拉取监测方案在指定时段的数据    │
  └─────┬────┘     │ 统计聚合、事件聚类、领域分类   │
        │          │ 敏感筛选、重点选题、素材打包    │
        ▼          └──────────────────────────────┘
  ┌──────────┐
  │ AI 生成   │ ──→ 调用大模型服务，传入预处理数据
  └─────┬────┘     大模型分章节并行生成
        │          返回完整报告 HTML
        ▼
  ┌──────────┐
  │ 预览编辑  │ ──→ 在线预览报告内容
  └─────┬────┘     支持在线编辑修改
        │          修改内容自动保存
        ▼
  ┌──────────┐
  │ 导出发布  │ ──→ 导出 Word / PDF
  └──────────┘     存入报告中心列表
```

| 页面 | 功能 |
|------|------|
| AI报告中心 | 报告模板列表 + 已生成报告列表 |
| 报告配置页 | 选择行业、填写基本信息、关联方案 |

## 七、关键词段段映射

报告各章节所需数据与 ES 字段的对应关系：

| 报告内容 | 所需字段 | 说明 |
|---------|---------|------|
| 信息总量 | 聚合 count | 按时间段统计 |
| 来源分布 | `g_pof`（舆论场） | 分舆论场统计 |
| 情感占比 | `n_sentiment_class` | 正面/中立/负面 |
| 领域分类 | `info_class` / `info_class_ai` | 行业领域标签 |
| 事件聚类 | `m_title` + `n_content_md5` + `y_content_simhash` | 标题相似度 + 去重 |
| 转载量 | `y_similar_count` | 相似文数量 |
| 热度评分 | `y_similar_count` × `n_media_level` × 时效性 | 综合评分 |
| 地域分布 | `n_content_areacode` | 内容提及地域 |
| 热搜关联 | `hot_search` / `hot_search_event` | 排名、在榜时长 |
| 媒体信息 | `n_media` / `n_media_level` / `m_site_name` | 媒体名称、等级 |
| 文章摘要 | `m_abstract` / `n_content_abstract` | 预提取的摘要 |
