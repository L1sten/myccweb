# 公安融媒宣传管理平台 - 产品需求文档

## 1. 产品概述

### 1.1 背景

公安机关在新媒体矩阵（抖音、微博、视频号、头条号等平台）运营大量官方账号，发布政务宣传内容后会收到大量用户评论。其中部分评论可能包含负面评价、质疑执法、讽刺批评等内容，若不及时发现和处置，可能引发舆情发酵。

### 1.2 目标

将公安矩阵账号的评论内容统一采集抓取，通过大模型智能识别负面评论，实现"采集→识别→预警→处置"的闭环管理。

- 评论来源一：网讯舆情系统的配置方案（微博、今日头条、抖音、微信等平台的关键词监测方案）
- 评论来源二：定向采集的视频号评论

### 1.3 用户角色

- **省级公安宣传管理员**：查看全省负面舆情和评论监测数据，统一管理和处置
- **市级公安宣传管理员**：查看本市范围内的评论数据，执行处置操作

## 2. 负面舆情（已有模块，本期两个变更点）

本模块为系统已有功能，此处仅记录本期变更内容。

### 2.1 变更一：舆情方案与矩阵账号微信渠道负责人绑定

- **变更前**：舆情信息必须与系统中的矩阵账号关联才会有数据进来
- **变更后**：舆情方案与矩阵账号微信渠道的运维负责人强关联，例如"台州公安"的舆情方案与"台州公安"微信账号的运维负责人绑定
- **影响范围**：数据接入规则，不影响页面展示和交互

### 2.2 变更二：舆情信息类型限定

- **变更内容**：舆情数据先采集入库，以列表形式展示相关信息
- **类型限定**：舆情信息的类型仅为主贴、转发，不含评论（评论归入评论监测模块处理）
- **影响范围**：数据采集范围和列表展示内容

## 3. 评论监测

### 3.1 页面结构

- 统计卡片行：监测评论总数、负面评论、未处理、已处理
- 平台筛选标签栏：全部、抖音、微博、视频号、头条号 + 右侧搜索框
- 数据表格：序号、平台、账号、正文内容、潜在负面点（AI 识别）、状态、操作
- 分页底栏

### 3.2 统计卡片

- **展示规则**：一行 4 个统计卡片，每个卡片包含图标、数量（大字号加单位"条"）、描述文字
  - 监测评论总数：蓝色图标，显示当前监测到的评论总量
  - 负面评论：红色图标+红色数字，显示经 AI 识别为负面的评论数量
  - 未处理：橙色图标，显示状态为"未处理"的评论数量
  - 已处理：绿色图标，显示状态为"已处理"的评论数量
- **交互规则**：点击卡片筛选对应状态的数据
  - 点击"监测评论总数"或"负面评论"：清除状态筛选，显示全部负面评论
  - 点击"未处理"：仅显示状态为"未处理"的评论
  - 点击"已处理"：仅显示状态为"已处理"的评论
  - 当前选中的卡片蓝色边框高亮
- **取数规则**：从评论监测数据库中读取统计数据，按状态分类汇总
- **数据筛选规则**：随卡片点击联动筛选；随平台筛选标签联动更新
- **数据排序规则**：无排序，纯统计数值
- **动态更新规则**：处置操作完成后自动刷新统计数据

### 3.3 平台筛选与搜索

- **展示规则**：左侧平台标签栏（全部/抖音/微博/视频号/头条号），当前选中标签蓝色高亮；右侧搜索输入框，placeholder 为"搜索评论内容…"
- **交互规则**：点击平台标签立即切换，高亮当前选中项，列表仅显示该平台评论；搜索框输入时实时过滤，匹配评论内容和账号名；平台筛选和搜索条件取交集
- **取数规则**：从评论监测数据库中读取评论数据，包含平台、账号、评论内容、AI 识别结果、评论时间等字段
- **数据筛选规则**：平台标签精确匹配评论来源平台（抖音/微博/视频号/头条号）；搜索框对评论内容和账号名进行模糊匹配（包含即可）；两个条件取交集
- **数据排序规则**：默认按评论时间倒序排列（最新评论排在最前）

### 3.4 数据表格

- **展示规则**：白色圆角表格，表头固定
  - 序号：自增编号，居中
  - 平台：平台图标+平台名称的组合标签，不同平台使用不同底色（抖音红色底、微博橙色底、微信绿色底、头条橙色底）
  - 账号：评论者昵称
  - 正文内容：评论原文文本，下方灰色小字显示评论时间（YYYY-MM-DD HH:mm 格式）
  - 潜在负面点：AI 识别结果标签，带"AI 识别"小徽章
    - 高级负面：红色标签（如"明确负面评价""质疑执法"）
    - 中级负面：橙色标签（如"表达不满/批评""质疑处理方式"）
    - 低级负面：黄色标签（如"负面价值判断""负面结果描述"）
  - 状态：评论处置状态标签，三种状态
    - 未处理：橙色标签
    - 已处理：绿色标签
    - 无需处理：灰色标签
  - 操作："详情""忽略""处置"三个文字按钮，"忽略"为灰色，"处置"为红色；状态为"未处理"时才显示"忽略"和"处置"按钮
- **交互规则**：
  - 点击"详情"跳转至该评论在原平台的原文页面（新窗口打开）
  - 点击"忽略"直接将该评论状态标记为"无需处理"
  - 点击"处置"弹出处置弹窗，复用舆情监测模块的处置逻辑
- **状态流转规则**：
  - 默认状态为"未处理"
  - 点击"忽略"按钮，状态从"未处理"变为"无需处理"
  - 处置弹窗点击"确定"后，状态从"未处理"变为"已处理"
  - 状态变更后同步更新列表和统计卡片数据
- **取数规则**：根据当前筛选条件从评论监测数据库中读取评论列表，每条记录包含平台、账号、评论内容、AI 识别的负面类型和等级、评论时间、处置状态、原文链接
- **数据排序规则**：按评论时间倒序排列

### 3.5 AI 负面识别说明

- **展示规则**：在"潜在负面点"列头显示"AI 识别"小徽章（紫色背景，白色"AI"字样）
- **识别流程**：评论采集后自动经过大模型分析，模型输出负面类型和等级，结果存入评论记录

- **负面类型分类表**（供大模型识别使用）：

| 等级 | 负面类型 | 说明 | 示例 |
|------|---------|------|------|
| 高 | 明确负面评价 | 对公安机关、警务人员或执法行为给出直接的负面评价和指责 | "这种处理结果真的让人心寒" |
| 高 | 质疑执法 | 质疑公安机关执法行为的合法性或合理性 | "为什么不抓那个女的" |
| 高 | 质疑处理方式 | 质疑公安机关对事件或案件的处理方式是否恰当 | "那婆娘怎么就放她走呢？" |
| 高 | 质疑执法程序 | 指出执法程序存在明显瑕疵或违反规范 | "执法程序存在明显瑕疵 应当启动内部调查" |
| 中 | 表达不满/批评 | 对公安机关的工作或行为表达不满情绪或批评意见 | "看了视频真的气不打一处来 这种执法方式必须要改" |
| 中 | 负面评价+讽刺 | 使用讽刺、嘲讽等修辞手法表达负面态度 | "又是在演戏给谁看呢 呵呵" |
| 中 | 质疑处理结果 | 对处理结果表示不认可或持怀疑态度 | "希望上级部门能重视这个问题 不要不了了之" |
| 中 | 质疑处理时效 | 质疑事件处理的速度或进度，表达等待焦虑 | "处理结果出来了吗？等了好久了" |
| 低 | 负面价值判断 | 对事件做出概括性的负面价值判断 | "害人害己" |
| 低 | 负面结果描述 | 描述事件带来的负面后果或影响 | "损失大了" |
| 低 | 负面情绪扩散 | 描述负面情绪的蔓延或对整体感受的消极表达 | "作为本地人听说了这件事 感觉很失望" |

## 4. 处置弹窗

### 4.1 触发条件

- 负面舆情 tab：点击表格操作列的"处置"按钮
- 评论监测 tab：点击表格操作列的"处置"按钮

### 4.2 展示规则

- 模态弹窗，居中显示，白色背景
- 标题栏：显示"处置"，标题下方有浅灰色分割线
- "处置结果"标签
- 单选按钮组（3 个选项横排）：
  - 已核查并处理该异常
  - 已核实并反馈上级部门
  - 已核实无此异常
- 多行文本输入框，placeholder 为"请输入处置结果"
- 右下角显示字数统计，格式为"当前字数 / 200"
- 文本框下方居中显示"暂无数据"提示文字（灰色）
- 底部操作按钮："取消"（灰色边框按钮）和"确定"（蓝色实心按钮）

### 4.3 交互规则

- 弹窗打开时，单选按钮默认未选中，文本框为空
- 单选按钮必须选择一项才可提交
- 文本框为可选填写，最大 200 字，实时更新字数统计
- 点击"取消"关闭弹窗，不保存任何操作
- 点击"确定"校验是否已选择处置结果：
  - 未选择：提示用户必须选择处置结果
  - 已选择：保存处置记录（包含处置结果选项、处置描述文本、处置时间、操作人），关闭弹窗，该条记录状态更新为"已处置"

### 4.4 数据规则

- **取数规则**：弹窗无需读取外部数据
- **数据存储规则**：确定提交后，将处置结果（选项文本）、处置描述（文本框内容）、处置时间（当前系统时间）、操作人（当前登录用户）写入该条评论/稿件的处置记录中
- **状态流转规则**：提交处置后，该条记录状态从"待处理"变为"已处置"

## 5. 全局规则

### 5.1 Tab 切换

- 页面顶部两个 Tab："负面舆情"和"评论监测"
- 点击 Tab 切换时，仅切换内容区域，侧边栏和顶部标题不变
- 当前选中 Tab 底部显示蓝色下划线

### 5.2 分页

- 所有列表均支持分页
- 分页底栏显示：总条数、每页条数下拉（10/20/50条/页）、页码按钮、跳转输入框
- 切换每页条数或页码后刷新列表

### 5.3 侧边栏导航

- 固定左侧，可展开/收起子菜单
- "舆情管理"子菜单展开后显示"舆情报告库"和"舆情监测"两个子项
- "新媒体矩阵管理"子菜单包含"内容巡检""矩阵管理""通知消息""内容管理"
- 当前页面高亮对应导航项，左侧显示蓝色竖条

### 5.4 响应式

- 当屏幕宽度小于 1200px 时，侧边栏宽度自动收窄
