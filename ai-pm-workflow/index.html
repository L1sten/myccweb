Chapter 03

需求调研：对话式采集

不再填模板，不再写长段。跟 Claude Code 像聊天一样说需求，它会主动追问关键维度，帮你补全盲点。

7 个关键维度自动评估：背景痛点、业务目标、用户场景、核心旅程、现有方案、业务规则、参考竞品
至少 3 轮深度追问，每轮 3-5 个启发式问题，挖掘你没说出口的需求
多轮确认后才开始写，确保方向对了再动手

Chapter 05

原型优化：Figma + AI 协作模式

Figma 每月 3000 积分有限，新模式：Figma 生成初稿 → Claude Code 读取优化，两者互补，各取所长。

旧模式
  • Figma 全程生成，积分飞速消耗
  • 生成后还要手动调整细节
  • 改一处要重新生成整个页面
新模式
  • Figma 出初稿，快速定方向
  • Claude Code 读取后精细优化
  • 增量修改，哪里不对改哪里
进阶技巧：安装 frontend-design / teach-impeccable 等 Skill，让 AI 自动套用专业设计规范 — 配色系统、字体排版、动效节奏、响应式策略一键提升，原型质量直接拉满
案例：舆情分析报告
Figma 生成仪表盘初稿，Claude Code 读取后优化图表布局、交互状态、数据展示细节
案例：云媒资部门素材库
Figma 生成编目功能框架，Claude Code 补充交互细节：表单验证、状态流转、空状态处理

Chapter 07

前后端协同：HTML + MD 就够了

不需要 Axure 链接、不需要 Figma 权限。给开发的交付物极其简单：

给前端
一个 HTML 文件
包含完整的 UI 和交互
.html
给后端
一个 MD 文件
包含接口逻辑和业务规则
.md
效果：开发不需要反复问"这个弹窗长什么样"、"这个状态是什么意思"，打开 HTML 就能看到所有细节。
测试协同 — 测试人员基于 PRD + 原型直接编写用例，异常分支和边界条件一目了然
测试视角：PRD 里的异常处理章节 + 原型里的空状态/错误提示，让测试同学不用反复找 PM 确认"断网了怎么办"、"没数据时长什么样"
效果：前后端测试三方拿到同一份交付物，无需反复找 PM 确认细节，协作效率翻倍提升。
附：GitHub Pages 部署 — git push 即可上线，外部合作方也能直接访问
Chapter 08

封装你的专属 Skill

当某个工作流步骤反复出现时，把它封装成 Skill，一劳永逸

你是否发现自己反复给 AI 输入同样的指令？比如"按这个格式写 PRD"、"用这个风格画原型"——这就是该封装 Skill 的信号。
skill-creator 是 Claude Code 内置的 Skill 创建工具，对话式引导你把重复指令打包成可复用的 Skill，一次封装、随处调用。
Markdown 格式 — Skill 本质是一个 .md 文件，易于编写和维护
自动安装 — 创建后自动注册到 Claude Code，立即可用

Chapter 09

Skills 生态：插件化的超能力

Claude Code 支持安装 Skills，像 VS Code 插件一样扩展 AI 的能力

专业前端设计规范，生成高质量 UI，避免 AI 味。内置配色、字体、动效、响应式等完整指南
prototype-prd-sync
原型与 PRD 双向同步，改了原型自动更新 PRD 中对应的交互描述和流程图
agile-pm-workflow

teach-impeccable
收集项目设计上下文，让 AI 生成的 UI 更贴合你的品牌和风格
figma 相关 Skills
Figma Code Connect、Design System Rules 等，打通设计到代码的最后一公里
myccweb-deploy
自动在 myccweb 创建项目并部署到 GitHub Pages，一个命令搞定发布

Takeaways

带走这 6 件事

01
原型即代码，代码即原型 — 别再拖控件了，用自然语言描述你的想法，AI 帮你还原成可运行的界面
02
PRD 不是 Word 文档 — HTML 格式的 PRD 可以直接嵌入原型、包含流程图、支持版本管理，开发打开就能看
03
Figma + AI 是互补，不是替代 — Figma 做创意发散，AI 做精细落地，两者结合效率最大化
04
给开发的交付物越简单越好 — 一个 HTML + 一个 MD，比 50 页 PPT 管用
05
Skills 是 AI 的插件系统 — 装上对的 Skills，Claude Code 就从通用助手变成你的专属 PM 助手
06
AI 只是加速了重复性工作的效率 — 省下来的时间，请用来思考业务底层逻辑和 AI 无法理解的事，比如商业化拓展
